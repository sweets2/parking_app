# F-36: Municipal Garages Layer

## Overview

Adds a toggleable layer showing Hoboken's 4 HPU-managed municipal parking garages as permanent map
markers. Useful when tow zones are active nearby and the user needs covered parking. The data is
fully static — 4 known addresses that rarely change — so there is no recurring fetcher. The creator
agent must geocode the 4 addresses via Nominatim at implementation time and bake the resulting
coordinates directly into `data/garages.json`.

**Nominatim requirement:** Node's built-in `fetch` requires an explicit `User-Agent` header or
Nominatim returns HTTP 403 (see CLAUDE.md F-21 gotcha). Use:
`"User-Agent": "hoboken-parking-app/1.0 (build-time geocoder)"`.

## Type (`shared/types.ts`)

```ts
export interface Garage {
  name: string;       // "Garage B"
  address: string;    // "28 2nd St"
  capacity: number;   // 829
  lat: number;
  lng: number;
  phone: string;      // "201-653-7333"
}
```

## Data File (`data/garages.json`)

The creator agent must call Nominatim for each address and record the returned `lat`/`lon`.
Source addresses (from hobokennj.gov/resources/municipal-garages):

| Name | Address | Capacity | Phone |
|------|---------|----------|-------|
| Garage B | 28 2nd St, Hoboken NJ | 829 | 201-653-7333 |
| Garage D | 215 Hudson St, Hoboken NJ | 488 | 201-653-7333 |
| Garage G | 315 Hudson St, Hoboken NJ | 488 | 201-653-7333 |
| Midtown Garage | 371 4th St, Hoboken NJ | 730 | 201-222-3621 |

Output schema: a plain JSON array of 4 objects matching `Garage`. All `lat`/`lng` values must
be in the Hoboken bounding box (lat 40.73–40.76, lng -74.05 to -74.02).

## `renderGarageMarkers` (`app/map.ts`)

**Module-level state to add** (alongside `_signLayers`, `_violationLayers`, etc.):

```ts
let _garageLayers: LeafletLayer[] = [];
let _garageMarkersVisible: boolean = true;
```

**Add to `initMap()` reset block** (after the existing `_violationHighlightsVisible = true` line):

```ts
_garageLayers = [];
_garageMarkersVisible = true;
```

**Function signature:**

```ts
export function renderGarageMarkers(garages: Garage[], visible: boolean): void
```

**Behaviour:**

1. Remove all existing layers in `_garageLayers` (`layer.remove()`), then reset `_garageLayers = []`
2. If `_map === null`, return
3. For each garage, create a `L.marker` with a `L.divIcon`:
   - `html`: `"🅿"` wrapped in a `<span>` with `class="garage-pin"`
   - `className`: `"sign-emoji-marker"`
   - `iconSize`: `[20, 20]`
   - `iconAnchor`: `[10, 10]`
4. Bind a popup with: garage name (`<strong>`), address, capacity as "N spaces", phone number
5. Set `_garageMarkersVisible = visible`
6. If `visible` is `true` and `_map !== null`, call `marker.addTo(_map)` for each marker
7. Push each marker to `_garageLayers`

**`setGarageMarkersVisible` signature:**

```ts
export function setGarageMarkersVisible(visible: boolean): void
```

Sets `_garageMarkersVisible = visible`. If `_map === null`, return. Iterates `_garageLayers`:
if `visible`, calls `layer.addTo(_map)`; otherwise calls `layer.remove()`.

## `main.ts` Integration

In `initBrowserApp()`, after the existing fire-and-forget fetches, add:

```ts
fetch("data/garages.json")
  .then((r) => r.json())
  .then((garages: Garage[]) => { renderGarageMarkers(garages, true); })
  .catch(() => { /* non-fatal */ });
```

Add the import for `renderGarageMarkers` and `setGarageMarkersVisible` from `"./map"`.
Add the import for `Garage` from `"../shared/types"`.

Wire the toggle button using the same `aria-pressed` pattern as `upcoming-toggle`:

```ts
const garageToggle = document.getElementById("garage-toggle");
garageToggle?.addEventListener("click", () => {
  const isOn = garageToggle.getAttribute("aria-pressed") === "true";
  const next = !isOn;
  setGarageMarkersVisible(next);
  garageToggle.setAttribute("aria-pressed", String(next));
  document.getElementById("garage-legend")?.classList.toggle("garage-off", !next);
  const status = document.getElementById("garage-legend")?.querySelector(".garage-status");
  if (status !== null && status !== undefined) {
    status.textContent = next ? "Enabled" : "Hidden";
  }
});
```

## Test Cases (`tests/unit/map.test.ts`)

Add a new `describe` block `"F-36 garage markers"` with:

```
GIVEN a garages array of 4 Garage objects and initMap() called,
WHEN renderGarageMarkers(garages, true) is called,
THEN mockMapInstance._layers contains 4 markers.

GIVEN garage markers have been rendered (visible = true),
WHEN renderGarageMarkers(garages, false) is called,
THEN mockMapInstance._layers contains 0 garage markers.

GIVEN renderGarageMarkers(garages, false) then renderGarageMarkers(garages, true),
WHEN checking mockMapInstance._layers,
THEN 4 markers are present on the map.

GIVEN an empty garages array,
WHEN renderGarageMarkers([], true) is called,
THEN no markers are added and no error is thrown.

GIVEN 4 garage markers rendered,
WHEN setGarageMarkersVisible(false) is called,
THEN all 4 markers are removed from mockMapInstance._layers.

GIVEN 4 garage markers hidden,
WHEN setGarageMarkersVisible(true) is called,
THEN all 4 markers are re-added to mockMapInstance._layers.
```

Use a `makeGarage()` helper:

```ts
function makeGarage(overrides: Partial<Garage> = {}): Garage {

    name: "Garage B",
    address: "28 2nd St",
    capacity: 829,
    lat: 40.736,
    lng: -74.034,
    phone: "201-653-7333",
    ...overrides,
  };
}
```
