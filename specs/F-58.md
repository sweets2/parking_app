# F-58 ‚ÄĒ Desktop Left-Panel Layout

## Purpose

At viewport widths ‚Č• 1024px, move the mode navigation, controls panels, and bottom sheet into a fixed 360px left sidebar so the layout matches the desktop prototypes (see `specs/prototypes/54067683-9579-45c2-9a43-3c2e51fb5dae.png` ‚ÄĒ Check desktop; `specs/prototypes/ea36cecd-2d50-463b-a6a6-fe4dd6e03502.png` ‚ÄĒ Current desktop). The map fills the remaining right portion of the viewport. Mobile and tablet layouts (< 1024px) are unchanged.

## Mode affected

Both.

## State fields read

None.

## State fields written

None.

## TypeScript contract signatures

None. This feature is HTML and CSS only ‚ÄĒ no TypeScript changes.

## DOM IDs/classes or visible copy

New element added to `index.html`:
- `#left-panel` ‚ÄĒ wrapper `<div>` enclosing `#mode-nav`, `#check-controls`, `#current-controls`, and `#bottom-sheet`

New CSS classes added to `style.css`:
- `#left-panel` ‚ÄĒ base styles (transparent/no-op on mobile) and desktop sidebar styles
- Desktop overrides on `#map`, `#mode-nav`, `#check-controls`, `#current-controls`, `.bottom-sheet`, `.bottom-sheet--open`, `#locate-control`, `#legend-stack`

No new visible copy.

## Behavior

### Mobile / tablet (< 1024px) ‚ÄĒ unchanged

All existing layout, animation, and interaction behavior is preserved exactly. `#left-panel` is invisible (no background, no border, no shadow) and its children flow as they did before being wrapped.

### Desktop (‚Č• 1024px)

**Left sidebar (`#left-panel`):**
- `position: fixed; left: 0; top: 0; bottom: 0; width: 360px`
- White background, right border (`1px solid #e5e7eb`), subtle box-shadow
- `display: flex; flex-direction: column`
- `z-index: 1100` (above map tiles, below modals/feedback overlay)
- `overflow-y: auto`

**Map (`#map`):**
- `margin-left: 360px; width: calc(100% - 360px)`
- Leaflet reads the updated container dimensions on initialization (CSS is applied before `DOMContentLoaded`)

**Mode nav (`#mode-nav`):**
- Reset 768px centering (`left: 50%`, `transform: translateX(-50%)`, `width: 320px`)
- `position: static; width: 100%; border-radius: 0`
- Flows as the first child of `#left-panel`
- Remove `border-left` and `border-right` that were added at 768px

**Check controls (`#check-controls`) and Current controls (`#current-controls`):**
- Reset 768px centering
- `position: static; width: 100%; border-radius: 0`
- Flow as the second child of `#left-panel` (one hidden at a time per existing JS logic)
- Remove `bottom` offset and side borders that were added at 768px

**Bottom sheet (`.bottom-sheet`):**
- `position: static; transform: none; flex: 1; overflow-y: auto`
- `border-radius: 0; box-shadow: none; max-height: none; border-top: 1px solid #e5e7eb`
- Flows as the last child of `#left-panel`
- When not open (`.bottom-sheet:not(.bottom-sheet--open)`): `display: none`
- When open (`.bottom-sheet--open`): visible in-panel (no slide animation needed; `transform: none` already set)

**Map overlay buttons ‚ÄĒ shift right to clear sidebar:**
- `#locate-control`: `left: 370px` (was `10px`)
- `#legend-stack`: `left: 370px` (was `10px`)
- `#feedback-btn` and `#coffee-btn` stay on the right edge ‚ÄĒ no change needed

## Acceptance criteria

Given a viewport width ‚Č• 1024px,
When the page loads,
Then `#left-panel` is rendered as a fixed 360px sidebar on the left, `#map` occupies the remaining viewport width to the right, and `#locate-control` / `#legend-stack` are not obscured by the sidebar.

Given a viewport width ‚Č• 1024px,
When the bottom sheet is closed (no `bottom-sheet--open` class),
Then `.bottom-sheet` has `display: none` and no slide-in animation occurs.

Given a viewport width ‚Č• 1024px,
When the bottom sheet is opened (class `bottom-sheet--open` is added),
Then `.bottom-sheet` becomes visible within the left panel with no upward translate animation and fills the available remaining height.

Given a viewport width < 1024px,
When the page loads,
Then the layout is identical to pre-F-58 behavior: `#map` fills the full viewport, controls are fixed at the bottom, and the bottom sheet slides up from the bottom.

Given a viewport width ‚Č• 1024px and the app is in Current mode,
When `#current-controls` is visible and `#check-controls` is hidden,
Then only `#current-controls` appears in the left panel (existing JS `display` toggling is unaffected).

## Preservation requirements

- All existing mobile layout and animations (bottom sheet slide-up, fixed bottom controls) are unchanged below 1024px.
- `showBottomSheet()` / `hideBottomSheet()` in `app/ui.ts` continue to add/remove `bottom-sheet--open` ‚ÄĒ no JS changes.
- Check ‚ÜĒ Current mode switching via `#check-controls` / `#current-controls` `display` toggling is unaffected.
- `#legend-stack`, `#locate-control`, `#coffee-btn`, `#feedback-btn`, and `#controls` remain functional.
- Leaflet map interaction (click, pan, zoom) is unaffected.
