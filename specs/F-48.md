# F-48 — Window-Based Schedule Evaluator

## What this is

Extends `shared/parking-logic.ts` with two pure functions that check whether a
parking restriction is active at any point during a given time window, rather
than at a single instant.

```typescript
/** Returns true if the recurring weekly schedule fires at any minute in [startTime, endTime]. */
export function isScheduleActiveInWindow(
  schedule: string,
  startTime: Date,
  endTime: Date,
): boolean

/** Returns true if the sign's [start_iso, end_iso] interval overlaps [startTime, endTime]. */
export function isSignActiveInWindow(
  sign: Sign,
  startTime: Date,
  endTime: Date,
): boolean
```

Files modified:

- `shared/parking-logic.ts` — two new exported functions appended after the
  existing `isScheduleUpcomingSoon`
- `tests/unit/parking-logic.test.ts` — new test cases added inside a new
  `describe("F-48 window evaluators")` block

## `shared/parking-logic.ts` — modifications

### `isScheduleActiveInWindow`

Implementation strategy: sample `[startTime, endTime]` in 30-minute steps and
call `isScheduleActiveNow` at each sample point. The maximum window the app
generates (overnight = ~10 h) yields at most 20 + 1 = 21 samples, which is
acceptable.

```typescript
export function isScheduleActiveInWindow(
  schedule: string,
  startTime: Date,
  endTime: Date,
): boolean {
  const STEP_MS = 30 * 60 * 1000; // 30-minute samples
  let t = startTime.getTime();
  const end = endTime.getTime();
  while (t <= end) {
    if (isScheduleActiveNow(schedule, new Date(t))) return true;
    t += STEP_MS;
  }
  return false;
}
```

**Contracts:**
- Uses existing `isScheduleActiveNow` — inherits all its Eastern-Time logic.
- For a zero-length window (`startTime === endTime`), behaves identically to
  `isScheduleActiveNow` at that instant.
- Returns `false` for unparseable schedules (same fallback as `isScheduleActiveNow`).

### `isSignActiveInWindow`

Interval-overlap test: `[signStart, signEnd]` overlaps `[startTime, endTime]`
iff `signStart ≤ endTime && signEnd ≥ startTime`.

```typescript
export function isSignActiveInWindow(
  sign: Sign,
  startTime: Date,
  endTime: Date,
): boolean {
  const signStart = new Date(sign.start_iso).getTime();
  const signEnd   = new Date(sign.end_iso).getTime();
  return signStart <= endTime.getTime() && signEnd >= startTime.getTime();
}
```

**Contracts:**
- Uses `sign.start_iso` / `sign.end_iso` (same fields as `isSignActive`).
- A sign that starts exactly at `endTime` is considered active (inclusive
  bounds on both sides).
- A sign that ends exactly at `startTime` is also considered active.

## Test list

All tests use the following fixtures from `tests/fixtures/signs.ts`:
- `NOW_STABLE` — a stable "now" reference in Eastern Time
- `NOW_AFTER_EXPIRED` — a "now" well past the expired-sign expiry

New fixture values (define inline in the test file; do not add to signs.ts):

```typescript
// A schedule active Monday–Friday 11 am – 1 pm ET
const WEEKDAY_SCHEDULE = "Monday through Friday   11 am – 1 pm";

// NOW_STABLE is a Monday at 11:30 am ET (it must be, per existing tests)
// so WEEKDAY_SCHEDULE is active at NOW_STABLE.
```

### `isScheduleActiveInWindow` tests

1. GIVEN `WEEKDAY_SCHEDULE` and a window that contains `NOW_STABLE` (e.g. NOW_STABLE ± 30 min),
   WHEN `isScheduleActiveInWindow` called,
   THEN returns `true`

2. GIVEN `WEEKDAY_SCHEDULE` and a window entirely outside the schedule (e.g. midnight to 9 am),
   WHEN `isScheduleActiveInWindow` called,
   THEN returns `false`

3. GIVEN `WEEKDAY_SCHEDULE` and a window that starts before and ends after the schedule window (11 am – 1 pm),
   WHEN `isScheduleActiveInWindow` called,
   THEN returns `true`

4. GIVEN `WEEKDAY_SCHEDULE` and a zero-length window equal to `NOW_STABLE`,
   WHEN `isScheduleActiveInWindow` called,
   THEN returns `true` (same as `isScheduleActiveNow`)

5. GIVEN an unparseable schedule string `""` and any window,
   WHEN `isScheduleActiveInWindow` called,
   THEN returns `false`

### `isSignActiveInWindow` tests

Use `SIGN_ACTIVE` and `SIGN_EXPIRED` fixtures from `tests/fixtures/signs.ts`.

6. GIVEN `SIGN_ACTIVE` and a window that overlaps its `[start_iso, end_iso]` range,
   WHEN `isSignActiveInWindow` called,
   THEN returns `true`

7. GIVEN `SIGN_ACTIVE` and a window entirely before `start_iso`,
   WHEN `isSignActiveInWindow` called,
   THEN returns `false`

8. GIVEN `SIGN_EXPIRED` (whose `end_iso` is before `NOW_STABLE`) and a window
   starting at `NOW_STABLE`,
   WHEN `isSignActiveInWindow` called,
   THEN returns `false`

9. GIVEN a sign whose `start_iso` equals `endTime` of the window,
   WHEN `isSignActiveInWindow` called,
   THEN returns `true` (inclusive end-boundary)

10. GIVEN a sign whose `end_iso` equals `startTime` of the window,
    WHEN `isSignActiveInWindow` called,
    THEN returns `true` (inclusive start-boundary)
