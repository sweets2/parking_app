## F-22 Popup Zoom-Resize Fix `[TODO]`

Bug fix: after opening a street-cleaning popup and zooming out, the popup shrinks proportionally (a Leaflet CSS transform side-effect that is accepted behaviour). Zooming back in leaves the popup at the smaller size; it does not resize until the user clicks elsewhere, which recreates the popup from scratch.

**Root cause:** Leaflet's internal zoom handling only calls `_updatePosition` on open popups (pixel repositioning). It does NOT call `_updateLayout` (width/height recalculation). After zoom-out, the popup's layout dimensions become stale relative to the new viewport context. On zoom-in, `_updatePosition` moves the popup again but `_updateLayout` is never called, so the popup stays at the constrained dimensions. Clicking elsewhere triggers `showStreetPopup`, which removes the old popup and creates a fresh one — going through Leaflet's full `onAdd → update()` path that resets both layout and position.

**Fix:** In `updateIconScale` (the `zoomend` handler inside `initMap`), call `popup.remove()` then `popup.openOn(map)` when a popup is open. `remove()` detaches the DOM without clearing `_latlng` or content. `openOn(map)` re-runs Leaflet's `onAdd`, which calls `popup.update()` internally and recalculates layout and position from scratch. The operation is synchronous — no visual blink.

---

### Module: `app/map.ts` — update `updateIconScale` inside `initMap`

Append at the end of the `updateIconScale` arrow function body, before its closing `}`:

```typescript
if (_streetPopup !== null) {
  _streetPopup.remove();
  _streetPopup.openOn(map);
}
```

No changes to any interface or exported symbol.
