## F-24 Road-Geometry Tow Segment Lines `[TODO]`

F-23 draws tow zone highlights by centering a straight polyline on `sign.lat`/`sign.lng` and orienting it E-W or N-S via street-name heuristic. Two visual problems: (1) the line sits at the curb/sidewalk, not the road surface; (2) diagonal streets like Observer Highway and curved streets get an incorrect straight bar.

F-24 replaces the heuristic with OSM road-centerline geometry fetched at build time, fixing both position and path. A casing style (white outer + red inner overlapping polylines) makes the highlight read as road-painted rather than floating.

Six files are added or modified:

- `fetcher/build-road-geometry.ts` ‚Äî new; Overpass query ‚Üí `data/road-geometry.json`
- `shared/types.ts` ‚Äî add `RoadGeometry` exported type
- `app/map.ts` ‚Äî update polyline shim; add `_roadGeometry`, `initRoadGeometry`, `getSubsegment`; update `renderTowSegments`; reset in `initMap`
- `app/main.ts` ‚Äî fire-and-forget fetch of `data/road-geometry.json`, call `initRoadGeometry`
- `package.json` ‚Äî add `build-road-geometry` script; extend `build` to copy file
- `tests/unit/map.test.ts` ‚Äî update polyline mock; update layer counts (√ó2 for casing); add geometry and style tests

### Module: `shared/types.ts` ‚Äî new export

Append after existing types:

```typescript
export type RoadGeometry = Record<string, [number, number][][]>;
```

Each key is a normalized street name (e.g. `"WASHINGTON ST"`). The value is an array of ways ‚Äî each way is an ordered array of `[lat, lng]` pairs. Separate arrays per way prevent phantom segments that would form at seams between geographically disjoint ways if naively concatenated.

### Module: `fetcher/build-road-geometry.ts` ‚Äî new file

Mirror the structure of `fetcher/geocode-cross-streets.ts`: named export `runBuildRoadGeometry()`, run-guard at the bottom using `process.argv[1]`.

#### Overpass query

Bounding box covers all of Hoboken:

```
[out:json][timeout:30];
way["highway"]["name"](40.734,-74.041,40.750,-74.025);
out geom;
```

Send `User-Agent: "hoboken-parking-app/1.0 (build-time road-geometry)"` header.

#### TypeScript types (strict-compatible ‚Äî no `as any`)

```typescript
interface OverpassNode { lat: number; lon: number; }
interface OverpassWay {
  type: string;
  tags?: Record<string, string>;
  geometry?: OverpassNode[];
}
interface OverpassResponse { elements: OverpassWay[]; }
```

#### Normalization

For each returned way:

1. Read `element.tags?.name` (skip if undefined)
2. Normalize to match sign-address format:
   - `name.toUpperCase()`
   - Word-boundary suffix replacements (regex with `\b`): `STREET‚ÜíST`, `AVENUE‚ÜíAVE`, `BOULEVARD‚ÜíBLVD`, `HIGHWAY‚ÜíHWY`, `PLACE‚ÜíPL`, `DRIVE‚ÜíDR`, `COURT‚ÜíCT`, `ROAD‚ÜíRD`, `TERRACE‚ÜíTER`
   - Directional word replacements (regex with `\b`): `NORTH‚ÜíN`, `SOUTH‚ÜíS`, `EAST‚ÜíE`, `WEST‚ÜíW`
3. Extract `element.geometry` as `[number, number][]` ‚Äî each node is `{ lat, lon }`, store as `[lat, lon]`
4. Append to `table[normalizedName]` (initialize as `[]` on first encounter)

#### Output

Write to `data/road-geometry.json` as `JSON.stringify(table, null, 2)`.

#### Run-guard pattern (mirror geocode-cross-streets.ts)

```typescript
const isMain =
  typeof process !== "undefined" &&
  Array.isArray(process.argv) &&
  typeof process.argv[1] === "string" &&
  (process.argv[1].endsWith("build-road-geometry.ts") ||
    process.argv[1].endsWith("build-road-geometry.js"));

if (isMain) {
  runBuildRoadGeometry().catch((err: unknown) => {
    console.error("Unhandled error:", err);
    process.exit(1);
  });
}
```

#### 6. New export: `getSubsegment(ways: [number, number][][], signLat: number, signLng: number): [number, number][]`

Pure geometry function ‚Äî no Leaflet calls.

**Fallback:** If `ways` is empty, or every way is empty, return the N-S 2-point fallback:

```typescript
[[signLat - 0.000162, signLng], [signLat + 0.000162, signLng]]
```

**Step 1 ‚Äî find nearest waypoint:**

For each way in `ways`, for each point in the way, compute flat-earth distance with cos(lat) correction:

```
dlat = (ptLat - signLat) * 111320
dlng = (ptLng - signLng) * 111320 * Math.cos(signLat * Math.PI / 180)
dist = Math.sqrt(dlat*dlat + dlng*dlng)
```

Track `bestWayIdx`, `bestPtIdx`, `bestDist`. The cos correction is required: at 40.74¬∞N, cos(40.74¬∞) ‚âà 0.757 ‚Äî without it, E-W distances are overestimated by ~32%, causing E-W subsegments to clip too short.

**Step 2 ‚Äî walk ¬±9 m from nearest waypoint:**

Walk backward (decreasing index) from `bestPtIdx` in `ways[bestWayIdx]`, accumulating waypoints until cumulative segment length ‚â• 9 m (using the same flat-earth formula). Walk forward (increasing index) with the same condition. In each direction, continue until the threshold is met or the end of the way is reached ‚Äî do not pad.

**Step 3 ‚Äî build result:**

```typescript
return [...backward.reverse(), bestPoint, ...forward];
```

If `result.length < 2` (degenerate: single-point way), return the N-S 2-point fallback centered on `bestPoint`.

#### 7. Update `renderTowSegments`

For each sign:

1. Extract street name: `sign.address.replace(/^\d[\d-]*\s+/, "").trim()`
2. Look up `_roadGeometry[streetName]`. If found and `ways.length > 0 && ways.some(w => w.length > 0)`, call `getSubsegment(ways, sign.lat, sign.lng)` for waypoints.
3. Else: fall back to the existing E-W/N-S 2-point heuristic (using `getStreetOrientation` ‚Äî logic unchanged from F-23).

**Draw two polylines per sign (casing) in both the geometry path and the fallback path ‚Äî push both to `_segmentLayers`:**

```typescript
const outer = L.polyline(waypoints, { color: "#fff", weight: 7, opacity: 0.65 });
const inner = L.polyline(waypoints, { color: "#cc0000", weight: 3, opacity: 0.85 });
_segmentLayers.push(outer, inner);
if (_towSignsVisible) {
  outer.addTo(_map);
  inner.addTo(_map);
}
```

The `setTowSignsVisible` logic already iterates over `_segmentLayers` to add/remove ‚Äî no changes needed there.

`_segmentLayers.length` is now `signs.length * 2`.

### Known gotchas (add to CLAUDE.md)

**F-24 ‚Äî `getSubsegment` longitude correction**
The flat-earth distance formula must apply cos(lat) to the longitude component: `dlng = (lng2-lng1) * 111320 * Math.cos(signLat * Math.PI/180)`. At 40.74¬∞N, cos(40.74¬∞) ‚âà 0.757; omitting this overestimates E-W distances by ~32%, causing E-W subsegments to clip short.

**F-24 ‚Äî separate way arrays prevent phantom segments**
`RoadGeometry` values are `[number, number][][]` (array of ways), not `[number, number][]` (flat). Concatenating all ways would create a phantom long segment at every seam between geographically disjoint ways. `getSubsegment` searches each way's points independently and returns a slice from the single nearest way.

**F-24 ‚Äî casing applies to both geometry and fallback paths**
Both `renderTowSegments` paths (geometry found and heuristic fallback) must draw 2 polylines per sign (outer white + inner red). The fallback does **not** keep the F-23 single red polyline. `_segmentLayers.length === signs.length * 2` in all cases.
