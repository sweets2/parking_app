### F-10.2 Tap to Save Spot

The user taps anywhere on the map to save their parking spot. `registerMapClickHandler` (from F-07) is wired in `main.ts`. In browsing mode, a tap immediately constructs a `SavedSpot` with the clicked coordinates and calls `app.onSaveSpot(spot)` — no intermediate button or street-side picker. There is no separate "position marker" step; the app transitions directly to parked mode.

**Tests**
- GIVEN the app is in browsing mode, WHEN the map click handler fires with (40.744, -74.032), THEN `app.onSaveSpot` is called with `{ lat: 40.744, lng: -74.032, ... }`.
- GIVEN the app is in browsing mode, WHEN the map click handler fires, THEN `renderPositionMarker` is NOT called (no intermediate position marker — the spot marker appears via the parked-mode `renderState` branch).

### F-10.3 Active Sign Pins Visible on Map

`app/map.ts` must export `signEmoji(reason: string): string` — a pure function returning an emoji character for a given sign reason. The `REASON_EMOJI` map in `app/map.ts` is the source of truth; `signEmoji` must wrap it. `renderSignPins` uses `L.divIcon` with the emoji to render each marker.

The `REASON_EMOJI` values in `app/map.ts` are:
- `"CONSTRUCTION"` → `"🚧"`
- `"MOVING"` → `"🚛"`
- `"EVENT"` → `"🎪"`
- `"DELIVERY"` → `"📦"`
- Unknown reason → `"⚠️"` (fallback)

**Tests**
- GIVEN `signEmoji("CONSTRUCTION")`, THEN it returns `"🚧"`.
- GIVEN `signEmoji("DELIVERY")`, THEN it returns `"📦"`.
- GIVEN `signEmoji("UNKNOWN_REASON")`, THEN it returns `"⚠️"` (fallback).
- GIVEN the app is in browsing mode with 3 active signs, WHEN `renderState` fires, THEN `renderSignPins` is called with an array of length 3 (mock-based: `vi.mock("../../app/map")`).
- GIVEN `app/map.ts` source, THEN `renderSignPins` contains popup-building code that references `sign.address` and `sign.reason`.

### F-10.4 Map-Click Auto-Save

Tapping the map in browsing mode is the only way to save a spot. There is no "Save My Spot" button and no street-side picker. The saved spot's `lat`/`lng` are exactly the tapped coordinates — no offset is applied.

**Tests**
- GIVEN the app is in browsing mode, WHEN the map is tapped at (40.744, -74.032), THEN `app.onSaveSpot` is called with `lat: 40.744` and `lng: -74.032`.
- GIVEN `app.onSaveSpot` is called, THEN the app transitions to `"parked"` state and `renderSpotMarker` is called (via `renderState` parked branch).
- GIVEN the app is closed and reopened after a spot was saved, THEN it opens in `"parked"` mode with the same spot (persisted in localStorage).
