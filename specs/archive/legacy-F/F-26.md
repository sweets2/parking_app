## F-26 Await Road Geometry Before Rendering Tow Segments `[TODO]`

`renderTowSegments` requires `_roadGeometry` to be populated to draw OSM-aligned polylines. In F-24/F-25, the road geometry fetch was fire-and-forget — `data/latest.json` (small) resolves before `data/road-geometry.json` (138 KB), so every page load calls `renderTowSegments` with an empty `_roadGeometry` and falls back to the N-S/E-W heuristic.

This feature converts the fire-and-forget road geometry fetch to an `await` so the data is guaranteed to be loaded before signs are rendered.

One file is modified:

- `app/main.ts` — convert road-geometry fetch from fire-and-forget to `await`

---

### Module: `app/main.ts` — modification

Replace:
```typescript
fetch("data/road-geometry.json")
  .then((res) => res.json())
  .then((data: RoadGeometry) => initRoadGeometry(data))
  .catch(() => { /* non-fatal: falls back to heuristic */ });
```

With:
```typescript
try {
  const geoRes = await fetch("data/road-geometry.json");
  initRoadGeometry(await geoRes.json() as RoadGeometry);
} catch { /* non-fatal: renderTowSegments falls back to heuristic */ }
```

This must appear before the `await fetch("data/latest.json")` call so geometry is available when signs are first rendered.

---

### Tests: `tests/unit/main.test.ts`

No new tests required — the existing main.test.ts suite already mocks `fetch` and verifies `renderTowSegments` call count. The mutation is non-observable from the test layer (the mock fetch resolves synchronously in tests regardless of await vs then-chain).
