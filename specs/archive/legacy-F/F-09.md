## F-09 UI Rendering Module `[TODO]`

`app/ui.ts` — pure DOM helpers. All functions accept data and modify the DOM. Owns no state. Does not touch the Leaflet map. Does not read from storage.

### Module interface — exported symbols

```typescript
export function renderLoading(): void
export function hideLoading(): void
export function renderSignCards(signs: Sign[], now: Date): void
export function renderBrowsingMode(activeSigns: Sign[], now: Date): void
export function renderClearBanner(): void
export function renderWarningBanner(nearbySigns: Sign[], now: Date): void

/** Duration the spot confirmation toast is visible before auto-hiding. */
export const TOAST_DURATION_MS = 4000

/** Briefly displays "Spot saved — [address]." (or just "Spot saved." if address is null) then hides after TOAST_DURATION_MS. */
export function showSpotToast(address: string | null): void
```

---

### F-09.1 Loading State

`renderLoading(): void` — inserts a spinner overlay. `hideLoading(): void` — removes it.

**Tests**
- GIVEN `renderLoading` is called, THEN a visible spinner element exists in the document body.
- GIVEN `hideLoading` is called after `renderLoading`, THEN the spinner is not in the DOM.

---

### F-09.2 Sign List Cards

`renderSignCards(signs: Sign[], now: Date): void` — renders a scrollable card list below the map. Each card shows: reason (badge), address, `formatSignWindow(sign, now)` (the active window countdown), full date range (`start_date` + `formatTime(start_time)` → `stop_date` + `formatTime(end_time)`), and permit number. Cards are sorted by `signSeverity`: high first, then medium, then low.

**Tests**
- GIVEN two active signs, WHEN `renderSignCards` is called, THEN two card elements exist in the sign list container.
- GIVEN a CONSTRUCTION and a DELIVERY sign, WHEN rendered, THEN the CONSTRUCTION card appears before the DELIVERY card in the DOM.
- GIVEN a sign with reason `"CONSTRUCTION"`, address `"259-265 11TH ST"`, and permit number `"637138"`, WHEN rendered, THEN the card contains all three strings.
- GIVEN an empty array, WHEN `renderSignCards` is called, THEN the sign list container is empty.
- GIVEN `renderSignCards` is called twice, THEN the second call replaces the first list (no duplicates).

---

### F-09.3 Browsing Mode UI

In browsing mode: show sign cards for all active signs. No status banner. There is no "Save My Spot" button — saving is triggered by tapping the map directly.

**Tests**
- GIVEN state is `"browsing"`, WHEN `renderBrowsingMode` is called, THEN no parked-state banner is in the DOM.
- GIVEN state is `"browsing"`, WHEN `renderBrowsingMode` is called, THEN the CLEAR MY SPOT and I'M HERE NOW buttons are hidden.

---

### F-09.4 Parked Mode — Clear Banner

`renderClearBanner(): void` — green banner reading "You're clear. No active signs near your spot."

**Tests**
- GIVEN `nearbySigns` is empty, WHEN `renderClearBanner` is called, THEN a green banner containing "clear" is visible.
- GIVEN the banner is shown, THEN the CLEAR MY SPOT and I'M HERE NOW buttons are visible.

---

### F-09.5 Parked Mode — Warning Banner

`renderWarningBanner(nearbySigns: Sign[], now: Date): void` — red or yellow banner. Shows the count of nearby active signs, the reason and address of the most severe, and a countdown from `formatCountdown`.

**Tests**
- GIVEN one nearby CONSTRUCTION sign with `end_iso: "2026-06-09T14:00:00"` (2 hours after `NOW_STABLE` in local time), WHEN `renderWarningBanner` is called with `now = NOW_STABLE`, THEN the banner contains `"CONSTRUCTION"`, the sign's address, and the exact string `"2h 0m"`.
- GIVEN two nearby signs, WHEN rendered, THEN the banner element contains the string `"2"`.
- GIVEN the most severe sign has severity `"high"`, THEN the banner element has `data-severity="high"`.
- GIVEN all nearby signs have severity `"medium"` or lower, THEN the banner element has `data-severity="medium"`.
- GIVEN `renderWarningBanner` is called with an empty array, THEN it does not throw and calls `renderClearBanner` instead (or equivalent safe fallback).

---

### F-09.7 Spot Confirmation Toast

After saving a spot, briefly display: "Spot saved — [address]." if an address is available, or "Spot saved." if address is null. The toast auto-hides after `TOAST_DURATION_MS` milliseconds.

**Tests**
- GIVEN `showSpotToast("259 11th St")` is called, WHEN the toast renders, THEN it contains `"259 11th St"`.
- GIVEN `showSpotToast(null)` is called, WHEN the toast renders, THEN it contains `"Spot saved"` and does not contain `"null"`.
- GIVEN the toast appears, THEN it is removed from the DOM after exactly `TOAST_DURATION_MS` milliseconds without user interaction.
