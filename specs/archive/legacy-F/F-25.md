## F-25 Side-Aware Perpendicular Offset for Tow Zone Polylines `[TODO]`

F-23/F-24 draw tow zone polylines on the OSM road centerline. When two signs restrict both sides of the same street block their polylines stack on top of each other. This feature offsets each line ±4 m toward its respective curb so opposite-side signs produce two visually distinct parallel lines.

The Hoboken tow sign API has no side field. Side is inferred from the sign's `lat`/`lng` position relative to the road centerline via a dot-product test against the perpendicular direction.

Two files are modified:

- `app/map.ts` — add `LATERAL_OFFSET_M` constant, add `offsetPolylinePoints` export, call it inside `renderTowSegments`
- `tests/unit/map.test.ts` — `offsetPolylinePoints` unit tests, two new `renderTowSegments` tests

#### `offsetPolylinePoints(pts, signLat, signLng, offsetM): [number, number][]` (exported named)

Shifts all points in `pts` perpendicular to the road direction toward the sign's curb side.

**Algorithm:**

1. Road direction vector in flat-earth metres from first → last point:
   ```
   cosLat = cos(signLat * π / 180)
   dY = (last[0] - first[0]) * 111320
   dX = (last[1] - first[1]) * 111320 * cosLat
   len = sqrt(dY² + dX²)
   ```
   If `len === 0` return `pts` unchanged.

2. Left-perpendicular unit vector (90° CCW from road direction in east/north space):
   ```
   perpX_m = -dY / len   (east component)
   perpY_m =  dX / len   (north component)
   ```

3. Sign displacement from segment midpoint in flat-earth metres:
   ```
   mid = [(first[0] + last[0]) / 2, (first[1] + last[1]) / 2]
   signDY = (signLat - mid[0]) * 111320
   signDX = (signLng - mid[1]) * 111320 * cosLat
   ```

4. Dot product determines side:
   ```
   dot = signDX * perpX_m + signDY * perpY_m
   ```
   - `|dot| < 1e-9` → sign is on centreline → return `pts` unchanged
   - `dot > 0` → sign is LEFT of road → `dir = +1`
   - `dot < 0` → sign is RIGHT of road → `dir = -1`

5. Apply uniform offset to every point:
   ```
   dLat = dir * offsetM * perpY_m / 111320
   dLng = dir * offsetM * perpX_m / (111320 * cosLat)
   pts.map(([lat, lng]) => [lat + dLat, lng + dLng])
   ```

**Heuristic fallback compatibility:** The fallback in `renderTowSegments` builds a 2-pt segment centred exactly on the sign, so the sign sits at the midpoint and `dot = 0`. `offsetPolylinePoints` returns `pts` unchanged for fallback segments — no existing tests are affected.

**Insertion point inside `renderTowSegments`:** Add the call immediately after the `if/else` block that assigns `waypoints` and before the `L.polyline` calls:
```typescript
waypoints = offsetPolylinePoints(waypoints, sign.lat, sign.lng, LATERAL_OFFSET_M);
```

### Tests: `tests/unit/map.test.ts`

#### New `describe("offsetPolylinePoints")` block (append after the last `describe("F-24 getSubsegment")` closing brace)

- GIVEN `pts.length < 2`, WHEN `offsetPolylinePoints` is called, THEN returns `pts` unchanged (degenerate guard)

- GIVEN a N-S road (two pts same lng, different lat) and sign east of centreline (`signLng > midLng`), WHEN called, THEN all returned points have `lng > original lng` (shifted east)

- GIVEN a N-S road and sign west of centreline (`signLng < midLng`), WHEN called, THEN all returned points have `lng < original lng` (shifted west)

- GIVEN an E-W road (two pts same lat, different lng) and sign north of centreline (`signLat > midLat`), WHEN called, THEN all returned points have `lat > original lat` (shifted north)

- GIVEN an E-W road and sign south of centreline (`signLat < midLat`), WHEN called, THEN all returned points have `lat < original lat` (shifted south)

- GIVEN sign exactly at midpoint (on centreline), WHEN called, THEN returns `pts` reference unchanged (dot ≈ 0 guard)

- GIVEN a N-S road and sign exactly `LATERAL_OFFSET_M` metres east of centreline, WHEN called with `offsetM = LATERAL_OFFSET_M`, THEN the east offset of each returned point ≈ `LATERAL_OFFSET_M` metres (use `toBeCloseTo(LATERAL_OFFSET_M, 1)`)

#### Two new tests inside `describe("F-23 renderTowSegments")`

- GIVEN two signs on opposite sides of the same N-S road (one east, one west), both injected via `initRoadGeometry`, WHEN `renderTowSegments` is called, THEN the east-side sign's polyline points all have `lng > centreline lng` and the west-side sign's polyline points all have `lng < centreline lng` (separation test — use the mock `polyline` to capture `_latlngs`)

- GIVEN a sign east of a N-S centreline, WHEN `renderTowSegments` is called, THEN all captured polyline points have identical `lat` values (offset is purely longitudinal — perpendicularity test)
