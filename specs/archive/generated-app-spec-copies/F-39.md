## F-39 Umami Analytics

Add privacy-first analytics via Umami Cloud (no cookies, no consent popup required).
A thin `app/analytics.ts` wrapper exposes a single `track()` function; every user-facing
interaction is instrumented across `main.ts`, `map.ts`, and `feedback.ts`.
A Vercel proxy rewrite (`/um/:path*`) recovers events dropped by ad-blockers.

`analytics.ts` guards with `typeof window !== "undefined"` so it is a silent no-op in
Node tests — no mocking of this module is needed in any existing test files.

Files created:
- `app/analytics.ts` — `track()` wrapper with Window interface extension
- `tests/unit/analytics.test.ts` — 4 test cases

Files modified:
- `app/index.html` — Umami script tag with placeholder IDs and TODO comment
- `vercel.json` — add rewrites proxy block
- `app/map.ts` — import track; debounced map-zoomed event
- `app/main.ts` — import track; `_lastAlertState` dedup var; 15 track call sites
  (includes 3 new toggle events added by F-36, F-37, F-38)
- `app/feedback.ts` — import track; feedback-opened and feedback-submitted events

### `app/analytics.ts` — new file

```typescript
declare global {
  interface Window {
    umami?: { track: (event: string, data?: Record<string, unknown>) => void };
  }
}

export function track(event: string, data?: Record<string, unknown>): void {
  if (typeof window !== "undefined") {
    window.umami?.track(event, data);
  }
}
```

The `typeof window !== "undefined"` guard is required. Tests run in Node
(`environment: "node"` in vitest.config) where `window` is undeclared — without
this guard the function throws `ReferenceError: window is not defined` at test time.

### `app/index.html` — modifications

Insert before `</head>` (after the `style.css` link):

```html
<!-- TODO before deploying: replace REPLACE_WITH_UMAMI_WEBSITE_ID and REPLACE_WITH_VERCEL_DOMAIN -->
<script
  defer
  src="/um/script.js"
  data-website-id="REPLACE_WITH_UMAMI_WEBSITE_ID"
  data-domains="REPLACE_WITH_VERCEL_DOMAIN"
  data-auto-track="false"
></script>
```

`data-auto-track="false"` disables Umami's built-in pageview tracking. This is a SPA;
all meaningful sessions are captured via the custom `app-loaded` event, so auto-tracked
pageviews would only add noise.

`data-domains` prevents localhost dev sessions and preview deploys from polluting the
production dashboard.

### `vercel.json` — modifications

Add a `rewrites` block to proxy Umami through the app domain. Ad-blockers that block
`cloud.umami.is` directly cannot block the app's own domain, so events are recovered.

Replace the file contents with:

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "app",
  "installCommand": "npm ci",
  "rewrites": [
    { "source": "/um/:path*", "destination": "https://cloud.umami.is/:path*" }
  ]
}
```

### `app/map.ts` — modifications

**Import** (add after existing imports):

```typescript
import { track } from "./analytics";
```

**Debounced map-zoomed** — inside `initMap()`, replace the single line:

```typescript
map.on("zoomend", updateIconScale);
```

with a closure that debounces the track call so rapid scroll-wheel zoom steps collapse
into one event (300ms window):

```typescript
let _zoomDebounce: ReturnType<typeof setTimeout> | null = null;
map.on("zoomend", () => {
  updateIconScale();
  if (_zoomDebounce !== null) clearTimeout(_zoomDebounce);
  _zoomDebounce = setTimeout(() => {
    track("map-zoomed", { zoom_level: map.getZoom() });
  }, 300);
});
```

`_zoomDebounce` is declared as `let` inside `initMap()` — scoped to the map instance,
not at module level — so it is reset on every `initMap()` call.

### `tests/unit/analytics.test.ts` — new file, 4 test cases

```
describe("analytics track()")
```

Test setup: `afterEach` deletes `(globalThis as Record<string, unknown>)["window"]`
to restore the Node environment between tests.

1. GIVEN the Node runtime has no window global (default state) WHEN `track("test-event")` is called THEN no exception is thrown and the call completes without error

2. GIVEN `(globalThis as Record<string, unknown>)["window"]` is set to an empty object (no umami property) WHEN `track("test-event")` is called THEN no exception is thrown

3. GIVEN `(globalThis as Record<string, unknown>)["window"]` is set to `{ umami: { track: vi.fn() } }` WHEN `track("map-zoomed", { zoom_level: 15 })` is called THEN `window.umami.track` is called exactly once with arguments `("map-zoomed", { zoom_level: 15 })`

4. GIVEN the same umami spy WHEN `track("app-loaded")` is called without a data argument THEN `window.umami.track` is called exactly once with arguments `("app-loaded", undefined)`

For tests 3 and 4: access the spy via the typed globalThis cast. Use `vi.fn()` and
`expect(spy).toHaveBeenCalledWith(...)` for assertions.

### Known gotchas

**`typeof window` vs bare `window` access** — The guard `typeof window !== "undefined"` is safe
in Node because `typeof` never throws on undeclared identifiers. A bare `window` reference would
throw `ReferenceError`. Never remove the `typeof` check.

**Test environment for window simulation** — In vitest's Node environment, setting
`(globalThis as Record<string, unknown>)["window"] = { ... }` makes bare `window` resolve to
that object because Node's module scope chains to `globalThis`. Clean up with `delete` in
`afterEach` to avoid cross-test pollution.

**Do not mock analytics in existing tests** — The `typeof window` guard makes `track()` a
no-op in Node. Existing `main.test.ts`, `map.test.ts`, and all other test files do not need to
mock `app/analytics` or add any analytics-related assertions.

**Toggle handler locations** — The garages, snow-routes, and bus-stops toggle callbacks
were added to `initBrowserApp()` by F-36, F-37, and F-38 respectively. Read the current
`app/main.ts` carefully to locate them — their exact position within the file depends on
the insertion order used by those features.
