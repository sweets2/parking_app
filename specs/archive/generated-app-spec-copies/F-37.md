# F-37: Snow Emergency Routes Layer

## Overview

Adds a toggleable layer showing Hoboken's designated snow emergency routes as blue street
polylines. During declared snow emergencies, parking is banned on these segments. The route
list is a small static dataset (8 segments) hardcoded from the city website.

The layer clips each route to its designated cross-street extents using optional `minLon`,
`maxLon`, `minLat`, `maxLat` fields on `SnowRoute`. For east-west streets use the lon fields;
for north-south streets (e.g. Willow Ave) use the lat fields. Without clip bounds the full OSM
geometry renders, which often extends past the designated endpoints. Uses blue (#3b82f6).

No real-time emergency detection in this spec. The layer is always shown when toggled on.

## Type (`shared/types.ts`)

```ts
export interface SnowRoute {
  street: string;   // road-geometry key, e.g. "3RD ST" (uppercase, no periods)
  side: string;     // "North" | "South" | "Both" | "West" | "East"
  from: string;     // cross-street label for display only
  to: string;       // cross-street label for display only
  minLon?: number;  // clip geometry west of this longitude (east-west streets)
  maxLon?: number;  // clip geometry east of this longitude (east-west streets)
  minLat?: number;  // clip geometry south of this latitude (north-south streets)
  maxLat?: number;  // clip geometry north of this latitude (north-south streets)
}
```

## `renderSnowEmergencyRoutes` (`app/map.ts`)

**Module-level state to add** (alongside `_signLayers`, `_violationLayers`, etc.):

```ts
let _snowRouteLayers: LeafletLayer[] = [];
let _snowRoutesVisible: boolean = true;
```

**Add to `initMap()` reset block** (after the `_garageMarkersVisible = true` line added by F-36):

```ts
_snowRouteLayers = [];
_snowRoutesVisible = true;
```

**Function signature:**

```ts
export function renderSnowEmergencyRoutes(routes: SnowRoute[], visible: boolean): void
```

**Behaviour — mirrors `drawStreetHighlight()` exactly, but with its own layer array:**

1. Remove all existing layers in `_snowRouteLayers` (`layer.remove()`), then reset
   `_snowRouteLayers = []`
2. Set `_snowRoutesVisible = visible`
3. If `_map === null`, return
4. For each `route` in `routes`:
   a. Look up `const ways = _roadGeometry[route.street]`
   b. If `ways` is `undefined` or `ways.length === 0`, skip (no error)
   c. For each `way` of `ways` (same loop as `drawStreetHighlight`):
      - If `way.length === 0`, skip
      - Create `L.polyline(way, { color: "#3b82f6", weight: 12, opacity: 0.45 })`
      - Push to `_snowRouteLayers`
      - If `_snowRoutesVisible` and `_map !== null`, call `layer.addTo(_map)`

**`setSnowRoutesVisible` signature:**

```ts
export function setSnowRoutesVisible(visible: boolean): void
```

Sets `_snowRoutesVisible = visible`. If `_map === null`, return. Iterates `_snowRouteLayers`:
if `visible`, calls `layer.addTo(_map)`; otherwise calls `layer.remove()`.

## `main.ts` Integration

In `initBrowserApp()`, after the existing fire-and-forget fetches, add:

```ts
fetch("data/snow-emergency-routes.json")
  .then((r) => r.json())
  .then(({ routes }: { routes: SnowRoute[] }) => {
    renderSnowEmergencyRoutes(routes, true);
  })
  .catch(() => { /* non-fatal */ });
```

Add imports for `renderSnowEmergencyRoutes`, `setSnowRoutesVisible` from `"./map"` and
`SnowRoute` from `"../shared/types"`.

Wire the toggle using the same `aria-pressed` pattern as `upcoming-toggle`:

```ts
const snowToggle = document.getElementById("snow-toggle");
snowToggle?.addEventListener("click", () => {
  const isOn = snowToggle.getAttribute("aria-pressed") === "true";
  const next = !isOn;
  setSnowRoutesVisible(next);
  snowToggle.setAttribute("aria-pressed", String(next));
  document.getElementById("snow-legend")?.classList.toggle("snow-off", !next);
  const status = document.getElementById("snow-legend")?.querySelector(".snow-status");
  if (status !== null && status !== undefined) {
    status.textContent = next ? "Enabled" : "Hidden";
  }
});
```

Note: the toggle calls `setSnowRoutesVisible` only — it does NOT re-call
`renderSnowEmergencyRoutes`. The layers are already in `_snowRouteLayers` from the initial
fetch render; toggling just adds or removes them from the map.

## Test Cases (`tests/unit/map.test.ts`)

Add a new `describe` block `"F-37 snow emergency routes"` with:

Use this helper:
```ts
function makeSnowRoute(overrides: Partial<SnowRoute> = {}): SnowRoute {
  return {
    street: "3RD ST",
    side: "North",
    from: "Jackson St",
    to: "River St",
    ...overrides,
  };
}
```

The tests below require `initRoadGeometry` to be called with geometry for matched streets.
Use the geometry key `"3RD ST"` with a simple 2-point way as the mock geometry.

```
GIVEN initMap() called, initRoadGeometry({ "3RD ST": [[[40.744, -74.032], [40.745, -74.032]]] }),
and routes = [makeSnowRoute({ street: "3RD ST" })],
WHEN renderSnowEmergencyRoutes(routes, true) is called,
THEN 1 polyline layer is present in mockMapInstance._layers with color "#3b82f6".

GIVEN snow routes have been rendered (visible = true),
WHEN renderSnowEmergencyRoutes(routes, false) is called,
THEN 0 polylines remain in mockMapInstance._layers.

GIVEN a route whose street key does not exist in road geometry,
WHEN renderSnowEmergencyRoutes([makeSnowRoute({ street: "NONEXISTENT ST" })], true) is called,
THEN no polyline is added and no error is thrown.

GIVEN geometry for "13TH ST" and two routes with street "13TH ST" but different sides
  ("North" and "South"),
WHEN renderSnowEmergencyRoutes(routes, true) is called,
THEN 2 polyline layers are present in mockMapInstance._layers (one per route entry per way).

GIVEN snow routes have been rendered,
WHEN setSnowRoutesVisible(false) is called,
THEN all snow polylines are removed from mockMapInstance._layers.

GIVEN snow routes have been hidden,
WHEN setSnowRoutesVisible(true) is called,
THEN all snow polylines are re-added to mockMapInstance._layers.
```
