# F-52 — Common Bottom Sheet Shell

## Purpose

Create a reusable bottom sheet shell for Check and Current mode detail panels.

## Mode affected

Both.

## State fields read

- activeMode
- selectedCheckSegment
- selectedRulesLocation
- rulesInspectionSections

## State fields written

None directly.

## TypeScript contract signatures

Update or create app/ui.ts exports:

```typescript
export function showBottomSheet(): void;
export function hideBottomSheet(): void;
export function setBottomSheetContent(content: string): void;
export function setBottomSheetMode(mode: AppMode): void;
```

## DOM IDs/classes or visible copy

Required DOM:
- #bottom-sheet
- #bottom-sheet-header
- #bottom-sheet-content
- #bottom-sheet-close

Required classes:
- .bottom-sheet
- .bottom-sheet--open
- .bottom-sheet--check
- .bottom-sheet--rules

## state/event/rendering behavior

Bottom sheet must:
- be closed by default
- open when content is selected or rendered
- not fully cover the map by default on mobile
- be usable at 375px width
- support Check and Current mode styling variants

## Acceptance criteria

Given the page has loaded and no function has been called
When the DOM is inspected
Then #bottom-sheet does NOT have the class .bottom-sheet--open.

Given showBottomSheet is called
When the DOM exists
Then #bottom-sheet receives .bottom-sheet--open.

Given hideBottomSheet is called
When the DOM exists
Then #bottom-sheet does not have .bottom-sheet--open.

Given setBottomSheetContent is called with the string "<p>hello</p>"
When the DOM exists
Then #bottom-sheet-content innerHTML equals "<p>hello</p>".

Given setBottomSheetMode("check") is called
Then #bottom-sheet has class .bottom-sheet--check and does not have .bottom-sheet--rules.

Given setBottomSheetMode("current") is called
Then #bottom-sheet has class .bottom-sheet--rules and does not have .bottom-sheet--check.

## Preservation requirements

- Preserve map visibility.
- Preserve Check | Current navigation.
- Preserve existing controls.
- Preserve all existing app/ui.ts exports unchanged: `renderLoading`, `hideLoading`, `renderSignCards`, `renderBrowsingMode`, `renderClearBanner`, `renderWarningBanner`, `showSpotToast`, `showStaleBanner`. Their signatures and behavior must not change.

## Behavior explicitly not implemented

- no saved-spot flow
- no reminder flow
- no notification flow
- no push flow
- no background monitoring
- no alert mode

## Non-goals

- No final Check content.
- No final Current mode content.
- No conflict evaluation changes.
