# F-55 — Rules Inspection UI / Bottom Sheet

## Purpose

Wire Rules mode map clicks to Rules inspection results, map marker/layers, and bottom sheet content.

## Mode affected

Rules.

## State fields read

- activeMode
- rulesTime
- parkingSegments
- selectedRulesLocation
- rulesInspectionSections

## State fields written

- selectedRulesLocation
- rulesInspectionSections

## TypeScript contract signatures

Use existing app/map.ts exports:

```typescript
setRulesInspectionMarker(lat: number, lng: number): void
renderRulesInspection(sections: RulesInspectionSection[]): void
clearRulesInspection(): void
```

Use shared/rules-inspector.ts:

```typescript
inspectRulesAtLocation(input: {
  lat: number;
  lng: number;
  selectedTime: Date;
  segments: ParkingSegment[];
}): RulesInspectionSection[]
```

## state/event/rendering behavior

When activeMode is "rules" and the user clicks the map:
1. Store selectedRulesLocation as `{ lat: number; lng: number }`.
2. Determine selectedTime from rulesTime.selectedTime.
3. Call `inspectRulesAtLocation({ lat, lng, selectedTime, segments: parkingSegments })`.
4. Store rulesInspectionSections (the returned array).
5. Call `setRulesInspectionMarker(lat, lng)`.
6. Call `renderRulesInspection(sections)`.
7. Render bottom sheet in Rules mode.
8. Bottom sheet lists all sections.

When activeMode switches away from "rules" (e.g., to "check"):
- Call `clearRulesInspection()` exactly once.

Bottom sheet content must include:
- selected location or nearest street when available
- each segment title (from section.title)
- side of street and location/range (encoded in section.title)
- active restriction summary (from section.content)
- next restriction when available (section.content will include "Next: <label> starting <ISO date>." if applicable)
- when no matching segment: section.title === "No matching segment" and section.content === "No parking segment found near this location."

## Acceptance criteria

Given activeMode is "rules"
When the map is clicked at lat 40.744, lng -74.032
Then selectedRulesLocation is set to `{ lat: 40.744, lng: -74.032 }`.

Given activeMode is "rules"
When the map is clicked
Then `inspectRulesAtLocation` is called once with `{ lat, lng, selectedTime: NOW_STABLE, segments: parkingSegments }` (use `NOW_STABLE` from `tests/fixtures/signs.ts` as the rulesTime.selectedTime in the test).

Given activeMode is "rules" and the map is clicked
When `inspectRulesAtLocation` returns two sections (e.g. two Washington St segments)
Then the bottom sheet renders exactly 2 section elements (e.g. elements with class `rules-section`), one per returned section.

Given activeMode switches from "rules" to "check"
When mode switch completes
Then `clearRulesInspection` has been called exactly 1 time.

Given no matching segment exists (inspectRulesAtLocation returns `[{ title: "No matching segment", content: "No parking segment found near this location.", priority: "unknown" }]`)
When the bottom sheet renders
Then the bottom sheet contains the text "No parking segment found near this location.".

Given `inspectRulesAtLocation` returns a section whose content includes "Next: Street cleaning starting 2026-06-10T14:00:00.000Z."
When the bottom sheet renders
Then the bottom sheet renders that "Next: ..." text verbatim within the section's content area.
