## F-20 Segment Auto-Highlight `[TODO]`

When a street cleaning popup opens, automatically detect which segment contains the tapped position and apply a visual highlight to that block. The popup appears immediately with all segments equal-weight (existing behavior); a background geocoding pass then re-renders the popup with one `sp-block--active` block highlighted.

The feature spans five files:
- `shared/parking-logic.ts` — two new pure helper functions
- `app/geo.ts` — new geocoding function with in-memory cache and shared rate limiting
- `app/style.css` — new highlight CSS class
- `app/map.ts` — optional `highlightedLocation` param on `buildStreetPopupContent`; async popup update with stale-token guard
- `app/main.ts` — `buildDetectSegmentCallback` factory; updated `showStreetPopup` call sites

---

### Module: `shared/parking-logic.ts` — new exports

#### `extractCrossStreets(location: string): [string, string] | null`

Parses a raw `StreetCleaningEntry.location` string into a two-element tuple of cross-street names.

- Split on the literal substring `" to "`.
- Expect exactly two parts after the split.
- Strip trailing `.` characters from each part (e.g. `"Observer Hwy."` → `"Observer Hwy"`).
- Return the tuple `[from, to]`, or `null` if the string does not contain `" to "` or produces a part count other than 2.

**Tests**
- GIVEN `"Observer Hwy. to Seventh St."`, THEN returns `["Observer Hwy", "Seventh St"]`.
- GIVEN `"9th St. to 10th St."`, THEN returns `["9th St", "10th St"]`.
- GIVEN `"Eighth St. and Tenth St."` (uses `" and "` not `" to "`), THEN returns `null`.
- GIVEN `"single street"` (no delimiter), THEN returns `null`.
- GIVEN `"A to B to C"` (three parts), THEN returns `null`.

#### `detectMatchingSegment(clickLat: number, clickLng: number, fromCoord: { lat: number; lng: number }, toCoord: { lat: number; lng: number }): boolean`

Returns `true` if the click coordinate falls between the two geocoded cross-street coordinates.

Algorithm:
1. Compute `deltaLat = Math.abs(fromCoord.lat - toCoord.lat)` and `deltaLng = Math.abs(fromCoord.lng - toCoord.lng)`.
2. If `deltaLat >= deltaLng`: street runs primarily N–S. Return `true` iff `clickLat` is in `[min(fromCoord.lat, toCoord.lat), max(fromCoord.lat, toCoord.lat)]` (inclusive).
3. Otherwise: street runs primarily E–W. Return `true` iff `clickLng` is in `[min(fromCoord.lng, toCoord.lng), max(fromCoord.lng, toCoord.lng)]` (inclusive).

**Tests**
- GIVEN a N–S street (deltaLat > deltaLng) and `clickLat` between the two endpoint latitudes, THEN returns `true`.
- GIVEN a N–S street and `clickLat` outside the latitude range, THEN returns `false`.
- GIVEN a N–S street and `clickLat` exactly equal to `fromCoord.lat` (boundary), THEN returns `true`.
- GIVEN a N–S street and `clickLat` exactly equal to `toCoord.lat` (boundary), THEN returns `true`.
- GIVEN an E–W street (deltaLng > deltaLat) and `clickLng` between the two endpoint longitudes, THEN returns `true`.
- GIVEN an E–W street and `clickLng` outside the longitude range, THEN returns `false`.
- GIVEN equal `deltaLat === deltaLng` (edge case), THEN does not throw and returns a boolean (treats as E–W).

### Module: `app/geo.ts` — new export

#### Module-level state (not exported)

Add two module-level declarations alongside the existing code:

```typescript
const _crossStreetCache = new Map<string, { lat: number; lng: number } | null>()
let _lastNominatimCallMs = 0
```

The `_lastNominatimCallMs` variable must also be written before every `fetch` call in the existing `getStreetName` function, so the 1 req/sec rate limit is enforced across both callers in this module.

#### Rate-limit helper (not exported)

Before each Nominatim fetch, compute `const elapsed = Date.now() - _lastNominatimCallMs`. If `elapsed < 1000`, insert `await new Promise<void>(r => setTimeout(r, 1000 - elapsed))` before the fetch. Set `_lastNominatimCallMs = Date.now()` immediately before each fetch call.

Apply this same pattern inside the existing `getStreetName` function as well, updating `_lastNominatimCallMs` there.

#### `geocodeCrossStreet(streetName: string): Promise<{ lat: number; lng: number } | null>` (exported named)

1. If `_crossStreetCache.has(streetName)`, return `_crossStreetCache.get(streetName) ?? null` immediately (no network call, no rate-limit wait).
2. Apply rate-limit delay as described above.
3. Set `_lastNominatimCallMs = Date.now()`.
4. Fetch: `https://nominatim.openstreetmap.org/search?format=json&q=${encodeURIComponent(streetName + ", Hoboken, NJ")}&limit=1`
   - Use an `AbortController` with an 8-second timeout, mirroring the pattern in `getStreetName`.
   - Include `Accept-Language: en` header.
5. Parse the JSON response as `Array<{ lat: string; lon: string }>`. If the array has at least one element, construct `{ lat: parseFloat(result[0].lat), lng: parseFloat(result[0].lon) }`.
6. On any error (network error, timeout, empty array, JSON parse failure), store `null` in the cache.
7. Store successful result in cache before returning.
8. Return the result or `null`.

**Tests** (append to `tests/unit/geo.test.ts`)
- GIVEN a mocked fetch returning `[{ lat: "40.744", lon: "-74.032" }]`, WHEN `geocodeCrossStreet("9th St")` is called, THEN returns `{ lat: 40.744, lng: -74.032 }`.
- GIVEN a mocked fetch returning an empty array `[]`, WHEN `geocodeCrossStreet("nonexistent")` is called, THEN returns `null`.
- GIVEN a mocked fetch that throws a network error, WHEN `geocodeCrossStreet("9th St")` is called, THEN returns `null` without throwing.
- GIVEN `geocodeCrossStreet("9th St")` has been called once successfully, WHEN it is called a second time, THEN fetch is NOT called again (returns the cached value).
- GIVEN `geocodeCrossStreet("bad")` returned `null` and was cached, WHEN it is called a second time, THEN fetch is NOT called again (null is cached too).

#### `showStreetPopup` — add optional `detectSegment` parameter

New signature:
```typescript
function showStreetPopup(
  lat: number,
  lng: number,
  streetName: string,
  entries: StreetCleaningEntry[],
  detectSegment?: (locations: string[]) => Promise<string | null>
): void
```

After opening the popup (existing behavior unchanged), add:
```typescript
const token = ++_popupHighlightToken
if (detectSegment !== undefined && entries.length > 0) {
  const uniqueLocations = [...new Set(entries.map(e => e.location))]
  void detectSegment(uniqueLocations).then((matched) => {
    if (_popupHighlightToken !== token) return
    if (_streetPopup === null) return
    if (matched === null) return
    _streetPopup.setContent(buildStreetPopupContent(streetName, entries, matched))
  })
}
```

The `void` keyword suppresses the floating-promise lint. Call sites that omit `detectSegment` continue to work without change.

**Tests** (append to `tests/unit/map.test.ts`)
- GIVEN `showStreetPopup` is called without a `detectSegment` argument, THEN the popup content does not contain the string `"sp-block--active"`.
- GIVEN `showStreetPopup` is called with a `detectSegment` that resolves to a specific location string, WHEN the promise resolves, THEN `popup.setContent` is called and the updated content contains `"sp-block--active"` exactly once.
- GIVEN `showStreetPopup` is called with a `detectSegment` that resolves to `null`, THEN `popup.setContent` is NOT called after the promise resolves.
- GIVEN `showStreetPopup` is called twice in rapid succession (second call increments token), WHEN the first call's `detectSegment` resolves last, THEN `popup.setContent` is NOT called for the first call (stale token).

### Module: `app/main.ts` — callback factory and call-site updates

#### `buildDetectSegmentCallback(clickLat: number, clickLng: number)`

A module-private function (not exported). Returns `(locations: string[]) => Promise<string | null>`.

The returned async function:
1. Iterates `locations` in order.
2. For each `location`, calls `extractCrossStreets(location)`. If it returns `null`, skip this location.
3. Calls `await geocodeCrossStreet(from)`. If it returns `null`, skip this location.
4. Calls `await geocodeCrossStreet(to)`. If it returns `null`, skip this location.
5. Calls `detectMatchingSegment(clickLat, clickLng, fromCoord, toCoord)`. If `true`, return `location`.
6. After all locations are exhausted, return `null`.

Requests are sequential (one `await` at a time) so the shared rate-limit clock in `geo.ts` works correctly.

#### Add new imports

```typescript
import { geocodeCrossStreet } from "./geo"
import { extractCrossStreets, detectMatchingSegment } from "../shared/parking-logic"
```

Parked-mode click handler (inside `registerMapClickHandler` callback):

const road = await getStreetName(lat, lng)
if (road !== null) {
