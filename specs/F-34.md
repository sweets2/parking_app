## F-34 Street Violation Highlights (Cleaning-Only)

Full-street semi-transparent shading layer: red for every street with an active
street-cleaning restriction right now, orange for streets that will be in
violation within the next 60 minutes. A legend toggle (same layout as the Tow
Zones legend) lets users hide the layer. Highlights update automatically at
each :00 hour boundary.

Tow zones already have their own distinct visual (red/white casing polylines).
The full-street shading layer covers **street cleaning only** — using both tow
signs here flooded the map with ~16 permanent permit signs painting nearly every
street red at all times.

Files modified:

- `app/map.ts` — `normalizeToGeometryKey`, `drawStreetHighlight`, `clearViolationHighlights`, `renderViolationHighlights`, `setViolationHighlightsVisible`
- `app/main.ts` — `scheduleViolationRefresh`, violation re-renders at sign state transitions, legend toggle wiring, initial render + schedule after `createApp()`
- `app/index.html` — add `#violation-legend` below `#tow-legend`
- `app/style.css` — add violation legend styles

### What "violation" means

Street-cleaning entries from `data/street-cleaning.json` are checked per refresh:

- `isScheduleActiveNow(entry.schedule, now)` → active (red)
- `isScheduleUpcomingSoon(entry.schedule, now)` → upcoming within 60 min (orange)

**Active takes priority**: draw orange only for streets that are upcoming but NOT active on that street.

**Street name normalisation** — `normalizeToGeometryKey(entry.street)` maps cleaning
entry street names to road-geometry keys:

- Uppercases, strips trailing periods (cleaning data uses "Adams St.", geo has "ADAMS ST")
- Expands abbreviations: STREET→ST, AVENUE→AVE, HIGHWAY→HWY, BOULEVARD→BLVD, PLACE→PL, COURT→CT, DRIVE→DR, ROAD→RD, TERRACE→TER
- Converts spelled-out ordinals to numeric: FIRST→1ST, SECOND→2ND, … SIXTEENTH→16TH
  (cleaning data uses "Eighth St.", geo has "8TH ST")

**Private helper `normalizeToGeometryKey`:**

```typescript
function normalizeToGeometryKey(name: string): string {
  return name
    .toUpperCase()
    .replace(/\./g, "")
    .replace(/\bSTREET\b/g, "ST")
    .replace(/\bAVENUE\b/g, "AVE")
    .replace(/\bHIGHWAY\b/g, "HWY")
    .replace(/\bBOULEVARD\b/g, "BLVD")
    .replace(/\bPLACE\b/g, "PL")
    .replace(/\bCOURT\b/g, "CT")
    .replace(/\bDRIVE\b/g, "DR")
    .replace(/\bROAD\b/g, "RD")
    .replace(/\bTERRACE\b/g, "TER")
    .replace(/\b(FIRST|SECOND|THIRD|FOURTH|FIFTH|SIXTH|SEVENTH|EIGHTH|NINTH|TENTH|ELEVENTH|TWELFTH|THIRTEENTH|FOURTEENTH|FIFTEENTH|SIXTEENTH)\b/g, m => ORDINAL_TO_NUMERIC[m] ?? m)
    .trim();
}
```

**Export `renderViolationHighlights`** (cleaning-only; no signs parameter):

```typescript
export function renderViolationHighlights(
  cleaningEntries: StreetCleaningEntry[],
  now: Date
): void {
  clearViolationHighlights();
  if (_map === null) return;

  const activeStreets = new Set<string>();
  const upcomingStreets = new Set<string>();

  for (const entry of cleaningEntries) {
    const street = normalizeToGeometryKey(entry.street);
    if (isScheduleActiveNow(entry.schedule, now)) {
      activeStreets.add(street);
    } else if (isScheduleUpcomingSoon(entry.schedule, now)) {
      upcomingStreets.add(street);
    }
  }

  for (const street of upcomingStreets) {
    if (!activeStreets.has(street)) {
      drawStreetHighlight(street, "#f97316", 0.22);
    }
  }
  for (const street of activeStreets) {
    drawStreetHighlight(street, "#ef4444", 0.28);
  }
}
```

### `tests/unit/map.test.ts` — test cases (7 total)

Fixtures:

```typescript
const ACTIVE_CLEANING: StreetCleaningEntry = {
  street: "Bloomfield Street",
  side: "West",
  schedule: "Monday through Friday   11 am – 1 pm",
  location: "1st St. to 14th St.",
};

const UPCOMING_CLEANING: StreetCleaningEntry = {
  street: "Bloomfield Street",
  side: "West",
  schedule: "Monday through Friday   1 pm – 2 pm",
  location: "1st St. to 14th St.",
};
```

1. Active cleaning schedule → red polylines on map
2. Upcoming cleaning schedule → orange polylines
3. Active and upcoming cleaning on same street → only red (active takes priority, no orange)
4. `setViolationHighlightsVisible(false)` → all violation layers removed from map
5. `setViolationHighlightsVisible(true)` → layers added back
6. `clearViolationHighlights` → all violation polylines removed from map
7. Empty road geometry → no error thrown, no polyline layers added
