# F-47 — Check Duration Bar + Simple Query Parser

## Purpose

Add Check mode duration controls and a pure parser for simple parking duration queries.

## Mode affected

Check.

## State fields read

- activeMode
- checkQuery

## State fields written

- checkQuery

## TypeScript contract signatures

Create shared/query-parser.ts.

Exports:

```typescript
export function parseCheckQuery(
  text: string,
  now: Date
): CheckQuery | null;

export function createDurationCheckQuery(
  minutes: number,
  now: Date
): CheckQuery;
```

## DOM IDs/classes or visible copy

Required UI:
- #check-controls
- #check-query-input
- #check-duration-30
- #check-duration-60
- #check-duration-120
- #check-until-input
- #check-run-button

Visible examples:
- 30 min
- 1 hour
- 2 hours
- Until
- Check parking

## state/event/rendering behavior

Only visible/active in Check mode.

Clicking duration buttons updates checkQuery.

Parser behavior:
- "30 min" means now through now + 30 minutes.
- "1 hour" means now through now + 60 minutes.
- "2 hours" means now through now + 120 minutes.
- "until 6pm" means now through next matching 6:00 PM.
- Unknown/unparseable text returns null.

No new Date() inside shared/query-parser.ts except through the injected now parameter.

## Acceptance criteria

Given now is NOW_STABLE (2026-06-09T16:00:00Z, which is 12:00 PM ET)
When createDurationCheckQuery(30, now) is called
Then startTime equals now, endTime equals now + 30 minutes (i.e. new Date(now.getTime() + 30 * 60 * 1000)), label is "30 min", and source is "duration".

Given now is NOW_STABLE
When createDurationCheckQuery(60, now) is called
Then startTime equals now, endTime equals now + 60 minutes, label is "1 hour", and source is "duration".

Given now is NOW_STABLE
When createDurationCheckQuery(120, now) is called
Then startTime equals now, endTime equals now + 120 minutes, label is "2 hours", and source is "duration".

Given now is NOW_STABLE
When parseCheckQuery("30 min", now) is called
Then it returns a CheckQuery with endTime equal to now + 30 minutes and source is "parser".

Given now is NOW_STABLE
When parseCheckQuery("1 hour", now) is called
Then it returns a CheckQuery with endTime equal to now + 60 minutes and source is "parser".

Given now is NOW_STABLE
When parseCheckQuery("2 hours", now) is called
Then it returns a CheckQuery with endTime equal to now + 120 minutes and source is "parser".

Given now is NOW_STABLE (12:00 PM ET on a weekday)
When parseCheckQuery("until 6pm", now) is called
Then it returns a CheckQuery with endTime equal to now + 360 minutes (6 hours) and source is "parser".

Given text is unsupported
When parseCheckQuery("banana", now) is called
Then it returns null.

Given activeMode is "rules"
When the page renders
Then the #check-controls element has the CSS property display set to "none".
