# F-46D — AppState Migration + Mode Switching + Layer Lifecycle Stubs

## Purpose

Migrate runtime app state to a ready-state model with activeMode and wire basic Check | Current mode switching. Add map layer lifecycle stubs required by later features.

## TypeScript contract signatures

AppState must become:

```typescript
type AppState =
  | { mode: "loading" }
  | { mode: "error"; message: string }
  | {
      mode: "ready";
      activeMode: AppMode;
      checkQuery: CheckQuery;
      checkResults: CheckResultSegment[];
      selectedCheckSegment: CheckResultSegment | null;
      rulesTime: RulesTimeSelection;
      selectedRulesLocation: { lat: number; lng: number; street?: string } | null;
      rulesInspectionSections: RulesInspectionSection[];
      allSigns: Sign[];
      activeSigns: Sign[];
    };
```

The `App` object returned by `createApp` must expose:

```typescript
export type App = {
  getState(): AppState;
  setActiveMode(mode: AppMode): void;
  tick(now: Date): void;
};
```

`setActiveMode` is a named method on the `App` object (not a standalone export from `main.ts` or `app.ts`). It:
- Updates `activeMode` in the ready state.
- When switching to `"check"`: calls `clearRulesInspection()` from `app/map.ts`.
- When switching to `"current"`: calls `clearCheckResults()` from `app/map.ts`.
- Is a no-op when called with the current active mode or when state is not `"ready"`.

`createApp` must accept `now: Date` as a third parameter so tests can inject a stable fixture time for `rulesTime.selectedTime`:

```typescript
export function createApp(
  deps: { renderState: (state: AppState) => void },
  initialData: { signs: Sign[]; fetchTime: Date },
  now: Date
): App
```

Note: `storage` and `SpotStorage` are removed from `deps` — this migration drops the saved-spot flow entirely.

Required `app/map.ts` exports (stubs — no real rendering yet):

```typescript
export function clearCheckResults(): void
export function renderCheckResults(results: CheckResultSegment[]): void
export function selectCheckSegment(id: string): void
export function clearRulesInspection(): void
export function renderRulesInspection(sections: RulesInspectionSection[]): void
export function setRulesInspectionMarker(lat: number, lng: number): void
export function showRulesControls(): void
export function hideRulesControls(): void
```

Map click routing in `app/main.ts`:

```typescript
function handleCheckClick(lat: number, lng: number): void
function handleRulesClick(lat: number, lng: number): void
```

Both are module-private functions (not exported). The single map click handler registered via `registerMapClickHandler` reads `app.getState()` to determine `activeMode` and dispatches to the appropriate function:
- `activeMode === "check"` → calls `handleCheckClick(lat, lng)`
- `activeMode === "current"` → calls `handleRulesClick(lat, lng)`

## state/event/rendering behavior

Legacy `browsing` and `parked` app states are removed. The `"ready"` state replaces both.

Initial ready state:
- `activeMode` defaults to `"check"`.
- `checkResults` defaults to `[]`.
- `selectedCheckSegment` defaults to `null`.
- `rulesTime` defaults to `{ mode: "now", selectedTime: now }` where `now` is the third parameter passed to `createApp`.
- `selectedRulesLocation` defaults to `null`.
- `rulesInspectionSections` defaults to `[]`.
- `allSigns` is set from `filterLoadTimeNoise(initialData.signs, initialData.fetchTime)`.
- `activeSigns` is set from `filterActive(allSigns, initialData.fetchTime)`.

Map click:
- Exactly one map click handler registered in `main.ts`.
- `activeMode === "check"` → calls `handleCheckClick(lat, lng)`.
- `activeMode === "current"` → calls `handleRulesClick(lat, lng)`.

## Existing app.test.ts migration

The following existing tests in `tests/unit/app.test.ts` must be **deleted** because the states and methods they test no longer exist:
- All tests that assert `state.mode === "browsing"` (legacy browsing mode is removed).
- All tests that assert `state.mode === "parked"` (legacy parked mode is removed).
- All tests calling `app.onSaveSpot(...)`, `app.onClearSpot()`, `app.onHereNow()`, `app.setUserPosition(...)` — these methods are removed from the `App` type.
- All tests exercising `userLat`, `userLng`, `spot`, or `nearbySigns` state fields.

The following must be **rewritten** with concrete expected values:
- The test for initial state: assert `state.mode === "ready"` and `state.activeMode === "check"` (not `"browsing"`).
- The tick test: after `app.tick(now)`, `state.mode` remains `"ready"` — not `"browsing"`.
- The `allSigns` test: `state.allSigns.length` is less than `ALL_SIGNS.length` (noise filtered), same assertion shape but under `state.mode === "ready"`.
