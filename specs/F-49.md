# F-49 — Choropleth Map Sweep Renderer

## What this is

Wires the query bar (F-46), time parser (F-47), and window evaluator (F-48)
together into a full color-coded map sweep.

When the user submits a query:
1. `parseQuery(text, now)` converts it to `{ startTime, endTime, label }`.
2. `renderSweepResults(cleaningEntries, signs, startTime, endTime)` in `map.ts`
   paints ALL road-geometry streets in three tiers:
   - **Red** (`#ef4444`, opacity 0.45) — a tow sign is active during the window
   - **Orange** (`#f97316`, opacity 0.30) — street cleaning fires during the window (no tow)
   - **Green** (`#22c55e`, opacity 0.20) — no restriction during the window
3. The query bar switches to its active state showing the label.

When the user clears the query:
1. `clearSweepResults()` removes the sweep polylines.
2. `renderViolationHighlights(cleaningEntries, now)` is called to restore
   real-time cleaning highlights.
3. The query bar returns to idle.

```typescript
export function clearSweepResults(): void {
  for (const layer of _sweepLayers) layer.remove();
  _sweepLayers = [];

```

### `renderSweepResults` (new export)

```typescript
export function renderSweepResults(
  cleaningEntries: StreetCleaningEntry[],
  signs: Sign[],
  startTime: Date,
  endTime: Date,
): void {
  clearSweepResults();
  clearViolationHighlights();
  if (_map === null) return;

  const L = getL();

  // Build a set of street keys with an active tow sign in the window.
  const towStreets = new Set<string>();
  for (const sign of signs) {
    if (!isSignActiveInWindow(sign, startTime, endTime)) continue;
    const rawStreet = sign.address.replace(/^\d[\d-]*\s+/, "").trim();
    towStreets.add(normalizeToGeometryKey(rawStreet));
  }

  // Build a set of street keys with active cleaning in the window.
  const cleaningStreets = new Set<string>();
  for (const entry of cleaningEntries) {
    if (!isScheduleActiveInWindow(entry.schedule, startTime, endTime)) continue;
    cleaningStreets.add(normalizeToGeometryKey(entry.street));
  }

  // Draw all streets in road geometry.
  for (const [streetKey, ways] of Object.entries(_roadGeometry)) {
    if (ways.length === 0) continue;
    const status: "tow" | "cleaning" | "safe" =
      towStreets.has(streetKey)     ? "tow"      :
      cleaningStreets.has(streetKey) ? "cleaning" :
      "safe";

    const color =
      status === "tow"      ? "#ef4444" :
      status === "cleaning" ? "#f97316" :
      "#22c55e";
    const opacity =
      status === "tow"      ? 0.45 :
      status === "cleaning" ? 0.30 :
      0.20;
    const weight = 6;

    const merged = mergeWays(ways);
    for (const way of merged) {
      const latlngs = way.map(([lat, lng]) => [lat, lng] as [number, number]);
      const poly = L.polyline(latlngs, { color, opacity, weight });
      poly.addTo(_map);
      _sweepLayers.push(poly as unknown as LeafletLayer);
    }
  }
}
```

**Contracts:**
- `mergeWays` is the existing private helper (already used by violation
  highlights). Call it here the same way.
- `normalizeToGeometryKey` is the existing private helper in `map.ts`.
- `isSignActiveInWindow` is imported from `../shared/parking-logic`.
- `isScheduleActiveInWindow` is imported from `../shared/parking-logic`.
- `clearViolationHighlights()` is called first to avoid rendering both
  real-time highlights and sweep results simultaneously.
- Safe-street polylines (green) are drawn for every key in `_roadGeometry` that
  has at least one way, including streets with no cleaning and no tow signs.
- Tow status takes priority over cleaning: a street that has both a tow sign
  AND a cleaning schedule active in the window renders red, not orange.

Replace the placeholder `onQuerySubmit` and `onQueryClear` functions (added in

are already in scope from the data-loading block at the top of `main()`.

```typescript
function onQuerySubmit(rawText: string): void {
  const query = parseQuery(rawText, new Date());
  if (query === null) {
    // Unrecognized — briefly flash the input red; do not switch to active state.
    const input = document.getElementById("query-input") as HTMLInputElement | null;
    if (input !== null) {
      input.style.color = "#ef4444";
      setTimeout(() => { input.style.color = ""; }, 800);
    }
    return;
  }
  setQueryActiveLabel(query.label);
  renderSweepResults(cleaningEntries, allSigns, query.startTime, query.endTime);
}

function onQueryClear(): void {
  setQueryIdle();
  clearSweepResults();
  renderViolationHighlights(cleaningEntries, new Date());
}
```

**Notes:**
- `allSigns` is the full sign array (already in scope; the same array passed to
  `renderSignPins` and `renderTowSegments`).
- `cleaningEntries` is the array from the street-cleaning JSON fetch (already in
  scope from F-34's main.ts wiring).
- `setQueryActiveLabel` and `setQueryIdle` are the helpers exported from main.ts
  in F-46.
- `renderViolationHighlights` is already imported from `./map` for the F-34
  scheduling logic.

```typescript
  street: "Adams Street",

  schedule: "Monday through Friday   11 am – 1 pm",  // active at NOW_STABLE
  location: "1st St. to 14th St.",
};

const WINDOW_END   = new Date(NOW_STABLE.getTime() + 30 * 60 * 1000);

// Window entirely outside the cleaning schedule (e.g. 3 am)

```

Test cases:

1. GIVEN road geometry with one street `"WASHINGTON ST"` and `ACTIVE_TOW_SIGN`
   (address "100 WASHINGTON ST") active in the window,
   WHEN `renderSweepResults([CLEANING_ENTRY], [ACTIVE_TOW_SIGN], WINDOW_START, WINDOW_END)` called,
   THEN at least one polyline is drawn with color `"#ef4444"` (red for tow)

2. GIVEN road geometry with `"ADAMS ST"` and `CLEANING_ENTRY` active in the window,
   WHEN `renderSweepResults([CLEANING_ENTRY], [], WINDOW_START, WINDOW_END)` called,
   THEN at least one polyline is drawn with color `"#f97316"` (orange for cleaning)

3. GIVEN road geometry with `"BLOOMFIELD ST"` and no cleaning or tow entries for it,
   WHEN `renderSweepResults([CLEANING_ENTRY], [], WINDOW_START, WINDOW_END)` called,
   THEN at least one polyline is drawn with color `"#22c55e"` (green for safe)

4. GIVEN a tow sign on `"WASHINGTON ST"` AND a cleaning entry for `"Washington Street"`,
   WHEN `renderSweepResults` called with a window that activates both,
   THEN `"WASHINGTON ST"` gets color `"#ef4444"` (tow takes priority over cleaning)

5. GIVEN `renderSweepResults` has been called once,
   WHEN `clearSweepResults()` called,
   THEN all sweep polylines are removed from the map (L.polyline mock shows no
   remaining layers — i.e., the `remove()` method was called on each added polyline)

6. GIVEN an empty road geometry `{}`,
   WHEN `renderSweepResults([CLEANING_ENTRY], [ACTIVE_TOW_SIGN], WINDOW_START, WINDOW_END)` called,
   THEN no error is thrown and no polylines are drawn

7. GIVEN `CLEANING_ENTRY` and `OFF_WINDOW_START/END` (window has no cleaning active),
   WHEN `renderSweepResults([CLEANING_ENTRY], [], OFF_WINDOW_START, OFF_WINDOW_END)` called
   with geometry for `"ADAMS ST"`,
   THEN the polyline drawn for `"ADAMS ST"` uses color `"#22c55e"` (safe, not orange)

turn green/orange/red based on overnight restrictions. Streets with tow zones

cleaning highlights.
