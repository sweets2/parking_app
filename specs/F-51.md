### renderCheckResults

`renderCheckResults` draws block/side-scoped Leaflet polylines for each evaluated segment.
Geometry is resolved from `_roadGeometry` (the module-level road geometry loaded via
`initRoadGeometry`) â€” **not** from `result.geometry` on `CheckResultSegment`. `result.geometry`
is a full-street lookup (`clipped: false`) and must not be used for Check rendering.

Geometry resolution per segment:
1. `normalizeToGeometryKey(result.street)` â†’ look up in `_roadGeometry`
2. `parseLocationBounds(result.location, streetKey)` â€” if null (location is not "X to Y" format),
   skip the segment silently (no polyline)
3. `clipWaysToBounds(allWays, bounds.min, bounds.max, bounds.axis)` â€” if empty after clip, skip
4. Apply curb-side offset via same zoom-adaptive logic as `drawWaysHighlight`

Signs (`Sign[]`) are passed so tow conflicts can use address-range subsegment geometry matching
`renderTowSegments`. `signs` defaults to `[]`.

Color/style map:

| status    | color    | opacity | dashArray | geometry source                          |
|-----------|----------|---------|-----------|------------------------------------------|
| `"safe"`  | `#22c55e`| 0.40    | none      | resolveCheckCurbWays                     |
| `"ticket"`| `#ef4444`| 0.60    | `"8,6"`   | resolveCheckCurbWays                     |
| `"tow"`   | `#dc2626`| 0.75    | none      | sign subsegment (if sign found) or resolveCheckCurbWays |
| `"unknown"`| `#94a3b8`| 0.40   | none      | resolveCheckCurbWays                     |
| `"snow"`  | â€”        | â€”       | â€”         | skipped (not produced by evaluateParkingWindow) |
| `"limited"`| â€”       | â€”       | â€”         | skipped (not currently produced)         |

For `"tow"` status: find first conflict with `sourceType === "tow-sign"`, look up the sign by
`conflict.sourceId`. If found, run the same address-range halfLengthM â†’ `getSubsegment` â†’
`offsetPolylinePoints` logic as `renderTowSegments`, storing white-casing + solid-red inner
polylines. If no matching sign, fall through to `resolveCheckCurbWays` + solid red polyline.

`renderCheckResults` calls `clearCheckResults()` at the top before drawing.

Side offset: use `result.side` when it is `"North"`, `"South"`, `"East"`, or `"West"`. Use
`null` (centerline) when `result.side` is `"Both"` or `"Unknown"`.

A module-private `Map<string, LeafletPolyline[]>` named `_checkLayers` stores id â†’ polylines.

### clearCheckResults

Removes all polylines from the map and empties `_checkLayers`.

### selectCheckSegment

Calls `setStyle({ className: "check-selected" })` on every polyline stored under that id in
`_checkLayers`. All other polylines are unaffected.

## Check legend

A `<div id="check-legend">` panel in `#legend-stack` (index.html) shows three swatches:
- Green solid â€” Safe
- Red dashed â€” Street cleaning
- Red solid â€” Tow zone

Visibility is toggled in `renderState()` in `main.ts` alongside the existing `checkControls` /
`rulesControls` blocks: `display = activeMode === "check" ? "" : "none"`.

## Caller contract

`check-controller.ts` passes `state.allSigns` as the second argument to `renderCheckResults`.
`main.ts` `renderState()` also passes `state.allSigns` when it calls `renderCheckResults`.

## state/event/rendering behavior

When the user runs a Check query:
- evaluate parking window for each parking segment
- store checkResults
- call `renderCheckResults(checkResults, state.allSigns)`

Selecting a segment:
- updates selectedCheckSegment
- calls `selectCheckSegment(id)`
- later bottom sheet features use this state

## Acceptance criteria

Given checkResults contains a safe segment with id "seg-safe" and a ticket segment with
id "seg-ticket", and road geometry is available with parseable location bounds for both
When renderCheckResults([safeSegment, ticketSegment]) runs
Then a green polyline (#22c55e) is drawn for "seg-safe" and a dashed red polyline (#ef4444,
dashArray "8,6") for "seg-ticket"; `_checkLayers` has size 2.

Given a segment whose street is not in _roadGeometry (or whose location cannot be parsed as
"X to Y")
When renderCheckResults([segment]) runs
Then no polyline is drawn and _checkLayers remains empty.

Given a "tow" segment whose conflict has a sourceId matching a sign in the signs array, and
road geometry has a matching street
When renderCheckResults([towSegment], [sign]) runs
Then 2 polylines are drawn (white casing + red inner) using sign subsegment geometry.

Given a segment with id "seg-ticket" has been rendered
When selectCheckSegment("seg-ticket") runs
Then setStyle is called with { className: "check-selected" } on that polyline.

Given activeMode switches to rules
When renderState runs
Then clearCheckResults() has been called and check-legend is hidden.
