# F-51 — Check Result Renderer

## Purpose

Render Check mode segment results on the map and allow selecting a segment.

## Mode affected

Check.

## State fields read

- activeMode
- checkResults
- selectedCheckSegment
- parkingSegments

## State fields written

- checkResults
- selectedCheckSegment

## TypeScript contract signatures

Upgrade the stubs from F-46D to real rendering in `app/map.ts`:

```typescript
export function clearCheckResults(): void
export function renderCheckResults(results: CheckResultSegment[]): void
export function selectCheckSegment(id: string): void
```

`renderCheckResults` draws a `L.polyline` for each non-safe segment using inline Leaflet style options (`color`, `weight`, `opacity`). Safe segments are **not drawn** — they produce no polyline and no `_checkLayers` entry. The color map is:

| ParkingStatus | color | opacity |
|---|---|---|
| `"ticket"` | `#ef4444` | 0.6 |
| `"tow"` | `#dc2626` | 0.7 |
| `"snow"` | `#3b82f6` | 0.55 |
| `"limited"` | `#f97316` | 0.55 |
| `"unknown"` | `#94a3b8` | 0.45 |

All polylines use `weight: 6`. The color/opacity values are passed as inline Leaflet options — **not** as a CSS `className`. `renderCheckResults` calls `clearCheckResults()` at the top before drawing.

`selectCheckSegment(id)` applies `setStyle({ className: "check-selected" })` to the polyline in `_checkLayers` for that id. All other polylines are unaffected.

A module-private `Map<string, L.Polyline>` named `_checkLayers` stores id → polyline. `clearCheckResults()` removes all entries from this map and removes each polyline from the map with `.remove()`.

## state/event/rendering behavior

When the user runs a Check query:
- evaluate parking window for each relevant segment
- store checkResults
- call renderCheckResults(checkResults)

Map result styling — each non-safe segment is drawn as a `L.polyline` with inline `color`/`weight`/`opacity` options from the table above. Safe segments are skipped entirely. The polyline uses the `geometry` field on `CheckResultSegment` for coordinates.

Selecting a segment:
- updates selectedCheckSegment
- calls selectCheckSegment(id)
- `selectCheckSegment(id)` looks up the polyline in `_checkLayers` by id and calls `setStyle({ className: "check-selected" })` on it
- later bottom sheet features use this state

## Acceptance criteria

Given checkResults contains a safe segment with id "seg-safe" and a ticket segment with id "seg-ticket"
When renderCheckResults([safeSegment, ticketSegment]) runs
Then `_checkLayers` has size 1 (only ticket is stored), the polyline for "seg-ticket" has color "#ef4444", and no polyline exists for "seg-safe".

Given a segment with id "seg-ticket" and status "ticket" has been rendered by renderCheckResults
When selectCheckSegment("seg-ticket") runs
Then setStyle is called on the "seg-ticket" polyline with { className: "check-selected" }.

Given activeMode is switched to rules
When mode switch completes
Then clearCheckResults() is called, which removes all polylines from the map and empties _checkLayers.
