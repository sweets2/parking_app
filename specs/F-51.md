# F-51 — Check Result Renderer

## Purpose

Render Check mode segment results on the map and allow selecting a segment.

## Mode affected

Check.

## State fields read

- activeMode
- checkResults
- selectedCheckSegment
- parkingSegments

## State fields written

- checkResults
- selectedCheckSegment

## TypeScript contract signatures

Upgrade the stubs from F-46D to real rendering in `app/map.ts`:

```typescript
export function clearCheckResults(): void
export function renderCheckResults(results: CheckResultSegment[]): void
export function selectCheckSegment(id: string): void
```

Each segment polyline rendered by `renderCheckResults` must be a `L.polyline` created with a `className` option drawn from the following map:

| ParkingStatus | className |
|---|---|
| `"safe"` | `"check-safe"` |
| `"unknown"` | `"check-unknown"` |
| `"limited"` | `"check-limited"` |
| `"ticket"` | `"check-ticket"` |
| `"tow"` | `"check-tow"` |
| `"snow"` | `"check-snow"` |

`selectCheckSegment(id)` must apply the CSS class `"check-selected"` to the polyline whose segment id matches, by calling `setStyle({ className: "check-selected" })` on that polyline layer. All other polylines retain their status-derived class.

A module-private `Map<string, L.Polyline>` named `_checkLayers` stores id → polyline. `clearCheckResults()` removes all entries from this map and removes each polyline from the map with `.remove()`.

## state/event/rendering behavior

When the user runs a Check query:
- evaluate parking window for each relevant segment
- store checkResults
- call renderCheckResults(checkResults)

Map result styling — each segment is drawn as a `L.polyline` with `className` set to its status class (see table above). The polyline uses the `geometry` field on `CheckResultSegment` for coordinates.

Selecting a segment:
- updates selectedCheckSegment
- calls selectCheckSegment(id)
- `selectCheckSegment(id)` looks up the polyline in `_checkLayers` by id and calls `setStyle({ className: "check-selected" })` on it
- later bottom sheet features use this state

## Acceptance criteria

Given checkResults contains a safe segment with id "seg-safe" and a ticket segment with id "seg-ticket"
When renderCheckResults([safeSegment, ticketSegment]) runs
Then `_checkLayers` has size 2, the polyline for "seg-safe" has className "check-safe", and the polyline for "seg-ticket" has className "check-ticket".

Given a segment with id "seg-safe" has been rendered by renderCheckResults
When selectCheckSegment("seg-safe") runs
Then setStyle is called on the "seg-safe" polyline with { className: "check-selected" }.

Given activeMode is switched to rules
When mode switch completes
Then clearCheckResults() is called, which removes all polylines from the map and empties _checkLayers.
