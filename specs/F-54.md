# F-54 — Rules Inspection Engine

## Purpose

Create pure domain logic that returns all relevant rules for a selected map location and time.

## Mode affected

Rules.

## State fields read

None.

## State fields written

None.

## TypeScript contract signatures

Create shared/rules-inspector.ts.

Required exports:

```typescript
export function inspectRulesAtLocation(input: {
  lat: number;
  lng: number;
  selectedTime: Date;
  segments: ParkingSegment[];
}): RulesInspectionSection[];

export function formatRuleSectionForSegment(input: {
  segment: ParkingSegment;
  selectedTime: Date;
}): RulesInspectionSection[];
```

## Behavior

### Proximity algorithm

`inspectRulesAtLocation` finds candidate segments using haversine distance from `(lat, lng)` to each segment's geometry centroid (average of all way coordinates). If a segment has no geometry, fall back to the midpoint derived from its location string if parseable, otherwise skip it.

- **Distance threshold:** 100 meters. Include all segments whose centroid is within 100 m of the clicked point.
- **Fallback:** If no segment is within 100 m, return a single "unknown" section (see acceptance criteria).
- **No deduplication by street name** — each segment is evaluated independently.

### formatRuleSectionForSegment output multiplicity

`formatRuleSectionForSegment` returns **exactly one** `RulesInspectionSection` per call. The returned array always has length 1.

The single section's fields:

- `title`: `"<street> (<side> side, <location>)"`. If side is "Unknown" omit the side clause: `"<street> (<location>)"`.
- `content`: A human-readable string summarising the active restriction(s) at `selectedTime`, or "No restriction at this time." when no restriction is active. When a `NextRestriction` exists, append `" Next: <label> starting <ISO date string>."` after the main content.
- `priority`: The highest `ParkingStatus` among active restrictions, or `"safe"` if none are active.

### Washington St hard rule

Do not merge multiple side/block-specific Washington St segments into one. If two segments both match the 100 m threshold and both have `street === "Washington Street"` (or street normalises to "WASHINGTON ST"), return a separate section for each — never collapse them.

### Unknown section shape

When no segment is within 100 m:

```
{ title: "No matching segment", content: "No parking segment found near this location.", priority: "unknown" }
```

### Priority

`priority` in a `RulesInspectionSection` maps directly to `ParkingStatus`: `"tow"` > `"snow"` > `"ticket"` > `"limited"` > `"unknown"` > `"safe"`. Use the highest-priority active restriction for the section's `priority`.

## Acceptance criteria

Given a location (lat, lng) whose nearest segment centroid is within 100 m
When inspectRulesAtLocation runs
Then it returns exactly 1 RulesInspectionSection whose `title` contains the segment's street name and whose `priority` matches the segment's active status at `selectedTime`.

Given a location is near two Washington St segments on different sides (e.g., East side and West side), both within 100 m
When inspectRulesAtLocation runs
Then it returns exactly 2 RulesInspectionSections, each with a distinct `title` that includes the side label (e.g., "East side" and "West side").

Given no segment centroid is within 100 m of the clicked location
When inspectRulesAtLocation runs
Then it returns exactly 1 RulesInspectionSection with `title === "No matching segment"`, `content === "No parking segment found near this location."`, and `priority === "unknown"`.

Given a segment whose towSigns array contains a sign active at selectedTime
When formatRuleSectionForSegment runs
Then the returned array has length 1 and `result[0].priority === "tow"`.

Given a segment with no restriction active at selectedTime but a tow sign starting after selectedTime
When formatRuleSectionForSegment runs
Then the returned array has length 1, `result[0].priority === "safe"`, and `result[0].content` contains the substring `"Next:"`.
