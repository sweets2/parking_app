## F-32 Tow Zone Popup Redesign `[DONE]`

Replaces the plain-text tow sign popup with a rich, styled bubble that mirrors the street-cleaning popup format. A prominent "🚨 TOW ZONE" header and a scary sub-line warn the user that towing is far more expensive than a street-cleaning ticket. The time window pulses red when a tow restriction is active now, and pulses amber when it starts within 60 minutes — identical animation curves to the street-cleaning pulse added in F-30.

Files modified:

- `app/map.ts` — import `isSignActive`, update `renderSignPins` to pass `now` to popup builder, rewrite `buildSignPopup`; add `closePopup()` to `LeafletMap` shim; call `_map.closePopup()` in `setTowSignsVisible` when hiding so open tow popups are dismissed
- `app/style.css` — add `.tz-*` component classes for the tow zone popup; reuse existing `schedule-breathe` / `schedule-warn` keyframes

---

### `app/map.ts` — modifications

**Import change**

Add `isSignActive` to the existing import from `../shared/parking-logic`.

**`renderSignPins(signs: Sign[], now: Date): void`**

Remove the `_` prefix from `now` (it was previously unused). Pass `now` to `buildSignPopup`.

**`buildSignPopup(sign: Sign, now: Date): string`**

Replaces the five-line plain-text implementation. Computes:

```typescript
const startMs = new Date(sign.start_iso).getTime();
const endMs   = new Date(sign.end_iso).getTime();
const nowMs   = now.getTime();

const active   = isSignActive(sign, now);
const upcoming = !active && startMs > nowMs
               && (startMs - nowMs) <= 60 * 60 * 1000
               && endMs > nowMs;
```

Reason label map:

| reason | label |
|---|---|
| `CONSTRUCTION` | `🏗 Construction` |
| `MOVING` | `🚚 Moving` |
| `EVENT` | `🎉 Event` |
| `DELIVERY` | `📦 Delivery` |

CSS reason modifier: `tz-reason--${sign.reason.toLowerCase()}`

Status row (only rendered when active or upcoming):
- Active: `<div class="tz-status tz-status--active">Active now — ends {stop_date} · {formatTime(end_time)}</div>`
- Upcoming: `<div class="tz-status tz-status--upcoming">Starts in {N}m</div>` where N = `Math.floor((startMs - nowMs) / 60_000)`

Full HTML structure:

```html
<div class="tz-wrap">
  <div class="tz-header">
    <span class="tz-icon">🚨</span>
    <span class="tz-title">TOW&nbsp;ZONE</span>
  </div>
  <div class="tz-sub">Towing costs hundreds — far more than a ticket</div>
  <hr class="tz-sep"/>
  <div class="tz-address">{sign.address}</div>
  <div class="tz-reason tz-reason--{reason}">{reasonLabel}</div>
  <!-- status row if active or upcoming -->
  <div class="tz-times">
    <div><span class="tz-time-label">From&nbsp;</span>{start_date} · {formatTime(start_time)}</div>
    <div><span class="tz-time-label">Until</span>{stop_date} · {formatTime(end_time)}</div>
  </div>
  <div class="tz-permit">Permit {permit_number}</div>
</div>
```
