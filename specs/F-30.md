## F-30 Schedule Violation Pulse Animation `[DONE]`

Street-cleaning schedule text in the popup pulses with a color animation when a violation is imminent or already active. Active schedules pulse red (1 s cycle); schedules starting within 60 minutes pulse amber-yellow (2 s cycle). Both states are bold.

Files modified:

- `shared/parking-logic.ts` — add `isScheduleActiveNow()` and `isScheduleUpcomingSoon()`
- `app/map.ts` — classify each schedule entry and apply the appropriate CSS class
- `app/style.css` — add `@keyframes` and `.sp-sched--active` / `.sp-sched--upcoming` classes

---

### `shared/parking-logic.ts` — additions

All time logic uses Eastern Time via `Intl.DateTimeFormat` with `timeZone: "America/New_York"`. No `new Date()` calls inside these functions — `now: Date` is always passed in.

#### Internal helpers (not exported)

**`parseScheduleHour(token: string): number`**
Parses a token like `"8 am"` or `"11 pm"` into a 24-hour integer. Returns `-1` on parse failure. Handles `12 am → 0` and `12 pm → 12`.

**`getEasternParts(now: Date): { dayIdx: number; minutesOfDay: number }`**
Uses `Intl.DateTimeFormat` with `weekday`, `hour`, `minute` parts. Returns `dayIdx` (0=Sun … 6=Sat, -1 on failure) and `minutesOfDay` (hours×60 + minutes, with `% 24` to normalise midnight "24").

**`parseScheduleRange(schedule: string): ScheduleRange | null`**
Splits on the triple-space separator (`"   "`). Day part supports single day (`"Monday"`) and range (`"Monday through Friday"`). Time part matches `"H am – H am"` pattern. Returns `null` on any parse failure.

```typescript
type ScheduleRange = {
  startDayIdx: number;
  endDayIdx: number;
  startMinutes: number;
  endMinutes: number;
};
```

#### Exported functions

**`isScheduleActiveNow(schedule: string, now: Date): boolean`**
Returns `true` if `dayIdx` is within `[startDayIdx, endDayIdx]` and `minutesOfDay` is within `[startMinutes, endMinutes)`.

**`isScheduleUpcomingSoon(schedule: string, now: Date): boolean`**
Returns `true` if `dayIdx` is within the day range and `minutesOfDay` is within `[startMinutes - 60, startMinutes)` — i.e. the schedule starts within the next 60 minutes.

### `app/map.ts` — modification

In `showStreetPopup` (or wherever schedule entries are rendered), for each `entry`:

```typescript
const schedActive   = now !== undefined && isScheduleActiveNow(entry.schedule, now);
const schedUpcoming = !schedActive && now !== undefined && isScheduleUpcomingSoon(entry.schedule, now);
const schedClass    = schedActive   ? "sp-sched sp-sched--active"
                    : schedUpcoming ? "sp-sched sp-sched--upcoming"
                    : "sp-sched";
```

`now` is passed in from the call site (never called with `new Date()` inside `map.ts`).

### Tests: `tests/unit/parking-logic.test.ts` — additions

Use `NOW_STABLE` and `NOW_AFTER_EXPIRED` from `tests/fixtures/signs.ts` where applicable. For time-sensitive assertions, construct explicit `Date` objects passed as `now`.

**`describe("isScheduleActiveNow")`**

- GIVEN schedule `"Monday   8 am – 11 am"`, WHEN `now` is Monday at 9:00 ET, THEN returns `true`
- GIVEN same schedule, WHEN `now` is Monday at 7:59 ET, THEN returns `false`
- GIVEN same schedule, WHEN `now` is Monday at 11:00 ET, THEN returns `false` (end is exclusive)
- GIVEN same schedule, WHEN `now` is Tuesday at 9:00 ET, THEN returns `false`
- GIVEN schedule `"Monday through Friday   8 am – 11 am"`, WHEN `now` is Wednesday at 9:00 ET, THEN returns `true`
- GIVEN unparseable schedule `"garbage"`, WHEN any `now`, THEN returns `false`

**`describe("isScheduleUpcomingSoon")`**

- GIVEN schedule `"Monday   8 am – 11 am"`, WHEN `now` is Monday at 7:30 ET (30 min before), THEN returns `true`
- GIVEN same schedule, WHEN `now` is Monday at 7:00 ET (exactly 60 min before), THEN returns `true`
- GIVEN same schedule, WHEN `now` is Monday at 6:59 ET (61 min before), THEN returns `false`
- GIVEN same schedule, WHEN `now` is Monday at 8:00 ET (schedule already started), THEN returns `false`
- GIVEN same schedule, WHEN `now` is Tuesday at 7:30 ET, THEN returns `false`
