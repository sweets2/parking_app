## F-18 Deployment Readiness `[TODO]`

Ensures the app can be built and deployed. This feature wires together the pieces that no single functional feature owns: the Leaflet CDN in `index.html`, the service worker compilation step in `package.json`, the PWA icon files, and the Vercel deploy config.

This feature modifies `app/index.html` and `package.json`. It also creates `app/icon-192.png`, `app/icon-512.png`, and `vercel.json`. It does not modify any TypeScript source files.

---

### Module interface — no new exported symbols

This feature produces no new TypeScript exports. All changes are to HTML, JSON, and config files.

---

### F-18.1 Leaflet CDN in index.html

`app/index.html` must load Leaflet before the app bundle so `L` is a global when `app.js` initializes. The service worker (`sw.ts`) pre-caches these same URLs:
- CSS: `https://unpkg.com/leaflet@1.9.4/dist/leaflet.css` — as a `<link rel="stylesheet">` in `<head>`
- JS: `https://unpkg.com/leaflet@1.9.4/dist/leaflet.js` — as a `<script>` tag in `<body>` before `app.js`

**Tests**
- GIVEN `app/index.html` source, THEN it contains a `<link>` tag whose `href` includes `leaflet` and `leaflet.css`.
- GIVEN `app/index.html` source, THEN it contains a `<script>` tag whose `src` includes `leaflet` and `leaflet.js`.
- GIVEN `app/index.html` source, THEN the Leaflet `<script>` tag appears before the `<script src="app.js">` tag (so `L` is defined when the module loads).

---

### F-18.2 Service Worker Build Step

`package.json` `"build"` script must compile `app/sw.ts → app/sw.js` using esbuild with `--platform=browser`. Without this step the service worker is never registered in production.

**Tests**
- GIVEN `package.json` source, THEN the `"build"` script contains `esbuild app/sw.ts`.
- GIVEN `package.json` source, THEN the `"build"` script contains `--outfile=app/sw.js`.
- GIVEN `package.json` source, THEN the `"build"` script contains `--platform=browser` (required for SW globals).

---

### F-18.3 PWA Icons

`manifest.json` references `icon-192.png` and `icon-512.png`. Both files must exist as valid PNG files in `app/`.

**Tests**
- GIVEN the `app/` directory, THEN `icon-192.png` exists and is a non-empty file.
- GIVEN the `app/` directory, THEN `icon-512.png` exists and is a non-empty file.
- GIVEN `app/icon-192.png`, THEN its first 8 bytes are the PNG signature `\x89PNG\r\n\x1a\n`.
- GIVEN `app/icon-512.png`, THEN its first 8 bytes are the PNG signature `\x89PNG\r\n\x1a\n`.

---

### F-18.4 Vercel Deploy Config

A `vercel.json` inside `generated_app/` tells Vercel how to build and where to serve from. The Vercel project's Root Directory must be set to `generated_app/` in the dashboard.

**Tests**
- GIVEN `vercel.json` source, THEN it contains `outputDirectory` set to `"app"` (the directory containing `index.html`).
- GIVEN `vercel.json` source, THEN it contains `buildCommand` that includes `npm run build`.
- GIVEN `vercel.json` source, THEN it contains `installCommand` set to `"npm ci"`.
