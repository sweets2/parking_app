# F-46 — Query Input Bar

## What this is

Adds a fixed query bar at the top of the map so users can type (or speak) a
parking intent like "overnight" or "right now". This is the UI shell only —
parsing and rendering are handled in F-47 and F-49.

The bar has two visual states:

- **Idle**: text input + mic button (mic hidden when Web Speech API is
  unavailable) + submit on Enter or click. Placeholder: `"Where should I park?
  (e.g. 'overnight')"`.
- **Active** (query in progress): replaces the input row with a label showing
  the raw query text and an `×` clear button. Tapping clear returns to idle
  and calls `onQueryClear()`.

`main.ts` registers two callbacks by wiring the DOM:
- `onQuerySubmit(rawText: string)` — called when the user submits a non-empty
  query. In F-49 this is replaced with the full sweep pipeline; here it is a
  no-op placeholder that stores the text and switches the bar to active state.
- `onQueryClear()` — called when the user taps `×`. Switches back to idle
  state.

Files modified:

- `app/index.html` — add `#query-bar` and `#query-active-bar` containers
- `app/style.css` — styles for both bar states
- `app/main.ts` — wire events; expose `setQueryActiveLabel(label: string)` and
  `setQueryIdle()` helpers so F-49 can drive the bar state

## `app/main.ts` — modifications

Add after other utility setup (after the locate-button wiring, before any
GeoWatch setup). Do not run at module scope — keep inside the `async function
main()` IIFE or equivalent top-level async block.

### Show/hide helpers (exported for F-49 to call)

```typescript
export function setQueryActiveLabel(label: string): void {
  const bar    = document.getElementById("query-bar");
  const active = document.getElementById("query-active-bar");
  const span   = document.getElementById("query-active-label");
  if (bar === null || active === null || span === null) return;
  span.textContent = label;
  bar.hidden    = true;
  active.hidden = false;
}

export function setQueryIdle(): void {
  const bar    = document.getElementById("query-bar");
  const active = document.getElementById("query-active-bar");
  const input  = document.getElementById("query-input") as HTMLInputElement | null;
  if (bar === null || active === null) return;
  active.hidden = true;
  bar.hidden    = false;
  if (input !== null) input.value = "";
}
```

### Event wiring

```typescript
function wireQueryBar(): void {
  const input     = document.getElementById("query-input")     as HTMLInputElement | null;
  const submitBtn = document.getElementById("query-submit-btn");
  const micBtn    = document.getElementById("query-mic-btn");
  const clearBtn  = document.getElementById("query-clear-btn");

  function submitQuery(): void {
    const text = input?.value.trim() ?? "";
    if (text.length === 0) return;
    onQuerySubmit(text);
  }

  input?.addEventListener("keydown", (e) => {
    if ((e as KeyboardEvent).key === "Enter") submitQuery();
  });
  submitBtn?.addEventListener("click", submitQuery);
  clearBtn?.addEventListener("click", () => onQueryClear());

  // Show mic button only if the browser supports Web Speech API
  const SpeechRecognition =
    (window as unknown as Record<string, unknown>)["SpeechRecognition"] as
    (new () => SpeechRecognitionInstance) | undefined ??
    (window as unknown as Record<string, unknown>)["webkitSpeechRecognition"] as
    (new () => SpeechRecognitionInstance) | undefined;

  if (SpeechRecognition !== undefined && micBtn !== null) {
    micBtn.hidden = false;
    let recognition: SpeechRecognitionInstance | null = null;
    micBtn.addEventListener("click", () => {
      if (recognition !== null) {
        recognition.stop();
        recognition = null;
        micBtn.classList.remove("recording");
        return;
      }
      recognition = new SpeechRecognition();
      recognition.lang = "en-US";
      recognition.interimResults = false;
      micBtn.classList.add("recording");
      recognition.onresult = (event: SpeechRecognitionEvent) => {
        const transcript = event.results[0]?.[0]?.transcript ?? "";
        if (input !== null) input.value = transcript;
        submitQuery();
      };
      recognition.onend = () => {
        micBtn.classList.remove("recording");
        recognition = null;
      };
      recognition.start();
    });
  }
}

// Placeholder callbacks — replaced in F-49
function onQuerySubmit(rawText: string): void {
  setQueryActiveLabel(`"${rawText}"`);
}

function onQueryClear(): void {
  setQueryIdle();
}

wireQueryBar();
```

### Type stub for Web Speech API (add near top of file)

```typescript
interface SpeechRecognitionEvent {
  results: { [index: number]: { [index: number]: { transcript: string } } };
}

interface SpeechRecognitionInstance {
  lang: string;
  interimResults: boolean;
  onresult: ((e: SpeechRecognitionEvent) => void) | null;
  onend: (() => void) | null;
  start(): void;
  stop(): void;
}
```

## Test list

1. GIVEN `wireQueryBar()` runs in a DOM environment with a text input and submit button,
   WHEN user types text and presses Enter,
   THEN `onQuerySubmit` is called with the trimmed text

2. GIVEN query input is empty,
   WHEN user presses Enter or clicks the submit button,
   THEN `onQuerySubmit` is NOT called

3. GIVEN a query has been submitted,
   WHEN `setQueryActiveLabel("overnight")` is called,
   THEN `#query-bar` is hidden and `#query-active-bar` is visible with the label text

4. GIVEN query active bar is showing,
   WHEN user clicks `#query-clear-btn`,
   THEN `onQueryClear` is called, `#query-active-bar` is hidden, `#query-bar` is visible

5. GIVEN `SpeechRecognition` is not present on `window`,
   WHEN `wireQueryBar()` runs,
   THEN `#query-mic-btn` remains hidden
