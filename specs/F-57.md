# F-57 — v2 UI Wiring: Mode Nav, Check Evaluation, Bottom Sheet

## Purpose

Wire the browser interactions that were left as stubs or partially connected after F-46D–F-56. All
logic and rendering infrastructure already exist; this feature connects the buttons to the state
machine and the evaluation pipeline to the map renderer.

## Mode affected

Check and Current.

## State fields read

- `activeMode`
- `parkingSegments`
- `rulesTime.selectedTime`

## State fields written

None via `app.ts`. Check results are held in a module-local `_checkResults` variable in `main.ts`
so `app.ts` does not need a `setCheckResults()` method.

## TypeScript contract signatures

No new exports. All changes are internal to `app/main.ts` and the `index.html` `hidden` attribute.

## DOM IDs/classes or visible copy

- `#query-bar` — add `hidden` attribute (vestigial from pre-v2 design; replaced by `#check-controls`)
- `.mode-nav-btn` — event listeners must call `app.setActiveMode(mode)` on click
- `.mode-nav-btn--active` — must be toggled in `renderState()` to match `state.activeMode`
- `#check-duration-30`, `#check-duration-60`, `#check-duration-120` — must set `aria-pressed` on the active button and clear the others
- `#check-query-input` — typing must clear the active duration button state
- `#check-run-button` — click must evaluate all `parkingSegments` and render results on the map
- `#bottom-sheet-close` — click must hide the bottom sheet


## Behavioral contracts

**Mode nav:**
- Clicking a `.mode-nav-btn` calls `app.setActiveMode(btn.dataset.mode)`.
- `renderState()` toggles `mode-nav-btn--active` on the matching button via `querySelectorAll`.
- The `querySelectorAll` call is guarded with `typeof document.querySelectorAll === "function"` so the test environment does not throw.

**Duration buttons:**
- Clicking a duration button stores the result of `createDurationCheckQuery(minutes, new Date())` into `_activeCheckQuery` (function-scoped inside `initBrowserApp`).
- The clicked button gets `aria-pressed="true"`; the other two get `aria-pressed="false"`.
- The `#check-query-input` value is cleared.
- Typing into `#check-query-input` clears `_activeCheckQuery` and resets all three buttons to `aria-pressed="false"`.

**Check run button:**
- Reads `_activeCheckQuery` (from a duration button click) or parses `#check-query-input` via `parseCheckQuery`.
- Falls back to `createDurationCheckQuery(120, new Date())` if both are null/empty.
- Evaluates: `state.parkingSegments.map(seg => evaluateParkingWindow(seg, resolvedQuery))`.
- Stores the result in `_checkResults`.
- Calls `clearCheckResults()` then `renderCheckResults(_checkResults)`.

**Bottom sheet close:**
- `#bottom-sheet-close` click calls `hideBottomSheet()`.

**renderState — Check results:**
- Uses `renderCheckResults(_checkResults)` instead of `renderCheckResults(state.checkResults)`.
- Does NOT call `clearCheckResults()` inside `renderState` (existing F-51 test asserts this).

## Acceptance criteria

Given the app is in ready/check mode, When a `.mode-nav-btn[data-mode="current"]` is clicked, Then `app.getState().activeMode` equals `"current"`.

Given the app is in ready/current mode, When a `.mode-nav-btn[data-mode="check"]` is clicked, Then `app.getState().activeMode` equals `"check"`.

Given `renderState` fires with `activeMode === "check"`, Then the `[data-mode="check"]` button has class `mode-nav-btn--active` and the `[data-mode="current"]` button does not.

Given the user clicks `#check-duration-30`, Then `#check-duration-30` has `aria-pressed="true"` and `#check-duration-60` has `aria-pressed="false"`.

Given the user has clicked a duration button and then types into `#check-query-input`, Then all three duration buttons have `aria-pressed="false"`.

Given `state.parkingSegments` is non-empty and the user clicks `#check-run-button`, Then `renderCheckResults` is called with an array whose length equals `state.parkingSegments.length`.

Given the bottom sheet is open and the user clicks `#bottom-sheet-close`, Then the bottom sheet no longer has class `bottom-sheet--open`.

Given `renderState` fires with `activeMode === "check"`, Then `clearCheckResults` is NOT called (rendering re-uses the previous `_checkResults` without clearing).
