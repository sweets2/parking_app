# F-53 — Rules Mode Time Selector + Layer Panel

## Purpose

Add Rules mode controls for selecting inspection time and preparing the layer panel UI.

## Mode affected

Rules.

## State fields read

- activeMode
- rulesTime

## State fields written

- rulesTime

## DOM IDs/classes or visible copy

Required DOM:
- #rules-controls
- #rules-time-now
- #rules-time-custom
- #rules-time-input
- #rules-layer-panel

Visible copy:
- Now
- Custom time
- Rules

## state/event/rendering behavior

Rules controls:
- visible/active only in Rules mode
- Now sets rulesTime.mode to "now"
- Custom time sets rulesTime.mode to "custom" and selectedTime to the chosen time

Layer panel:
- may be simple initially
- must not imply alerting or background monitoring

## Acceptance criteria

Given activeMode is "rules"
When the page renders
Then Rules controls are visible.

Given activeMode is "check"
When the page renders
Then Rules controls are hidden or inactive.

Given Now is selected
When state is read
Then rulesTime.mode is "now".

Given Custom time is selected
When a valid time is entered
Then rulesTime.mode is "custom" and selectedTime is updated.

## Preservation requirements

- Preserve Check controls.
- Preserve bottom sheet shell.
- Preserve map-first layout.

## Behavior explicitly not implemented

- no saved-spot flow
- no reminder flow
- no notification flow
- no push flow
- no background monitoring
- no alert mode

## Non-goals

- No Rules inspection engine.
- No Rules bottom sheet content.
- No map click inspection yet.
