# F-38: Bus Stops Layer

## Type (`shared/types.ts`)

```ts
export interface BusStop {
  id: string;    // stop_id from GTFS stops.txt
  name: string;  // stop_name from GTFS stops.txt
  lat: number;   // stop_lat
  lng: number;   // stop_lon
}
```

## Fetcher (`fetcher/fetch-bus-stops.ts`)

A build-time Node script. Steps:

1. Download the GTFS ZIP:
   ```
   https://cdn.transit.land/feeds/f-dr5-nj~transit~bus/latest.zip
   ```
   Set `"User-Agent": "hoboken-parking-app/1.0 (build-time bus-stop fetcher)"` on the request
   (Node's `fetch` requires an explicit User-Agent for Transitland â€” see CLAUDE.md F-21 gotcha).

2. Unzip in memory using the `fflate` package (already in devDependencies via the GTFS workflow),
   or use `node:zlib` + manual ZIP parsing. Read `stops.txt` as a UTF-8 string.

3. Parse `stops.txt` as CSV:
   - Split into lines; use `lines[0]` as the header row
   - Find column indices **by name** (not by assumed position):
     ```ts
     const headers = lines[0].split(",");
     const idIdx   = headers.indexOf("stop_id");
     const nameIdx = headers.indexOf("stop_name");
     const latIdx  = headers.indexOf("stop_lat");
     const lonIdx  = headers.indexOf("stop_lon");
     ```
   - Handle quoted fields: some `stop_name` values contain commas inside `"..."`. Use a simple
     CSV field parser that respects double-quote wrapping.
   - Skip any data line where parsed `lat` or `lng` is `NaN`.

4. Filter to Hoboken bounding box:
   ```
   lat >= 40.733 && lat <= 40.756 && lng >= -74.042 && lng <= -74.024
   ```

5. Write `data/bus-stops.json` as a plain `BusStop[]` array (not wrapped in an object).

6. Log: `"Wrote N bus stops to data/bus-stops.json"`

**`package.json` script to add:**

```json
"fetch-bus-stops": "npx tsx fetcher/fetch-bus-stops.ts"
```

## `renderBusStopMarkers` (`app/map.ts`)

**Module-level state to add:**

```ts
let _busStopLayers: LeafletLayer[] = [];
let _busStopsVisible: boolean = false;
```

**Add to `initMap()` reset block** (after the `_snowRoutesVisible = true` line added by F-37):

```ts
_busStopLayers = [];
_busStopsVisible = false;
```

**Function signature:**

```ts
export function renderBusStopMarkers(stops: BusStop[], visible: boolean): void
```

**Behaviour:**

1. Remove all existing layers in `_busStopLayers` (`layer.remove()`), then reset
   `_busStopLayers = []`
2. Set `_busStopsVisible = visible`
3. If `_map === null`, return
4. For each stop, create a `L.marker` with a `L.divIcon`:
   - `html`: `"đźšŚ"`
   - `className`: `"sign-emoji-marker"`
   - `iconSize`: `[14, 14]`  (smaller than garage markers)
   - `iconAnchor`: `[7, 7]`
5. Bind popup with `stop.name` only
6. If `visible` is `true`, call `marker.addTo(_map)` for each marker
7. Push each marker to `_busStopLayers`

**`setBusStopsVisible` signature:**

```ts
export function setBusStopsVisible(visible: boolean): void
```

Sets `_busStopsVisible = visible`. If `_map === null`, return. Iterates `_busStopLayers`:
if `visible`, calls `layer.addTo(_map)`; otherwise calls `layer.remove()`.

## `main.ts` Integration

In `initBrowserApp()`, after the existing fire-and-forget fetches, add:

```ts
fetch("data/bus-stops.json")
  .then((r) => r.json())
  .then((stops: BusStop[]) => { renderBusStopMarkers(stops, false); })
  .catch(() => { /* non-fatal */ });
```

Add imports for `renderBusStopMarkers`, `setBusStopsVisible` from `"./map"` and
`BusStop` from `"../shared/types"`.

Wire the toggle button. Note: `aria-pressed` starts `"false"` (layer is off by default):

```ts
const busToggle = document.getElementById("bus-toggle");
busToggle?.addEventListener("click", () => {
  const isOn = busToggle.getAttribute("aria-pressed") === "true";
  const next = !isOn;
  setBusStopsVisible(next);
  busToggle.setAttribute("aria-pressed", String(next));
  document.getElementById("bus-legend")?.classList.toggle("bus-off", !next);
  const status = document.getElementById("bus-legend")?.querySelector(".bus-status");
  if (status !== null && status !== undefined) {
    status.textContent = next ? "Enabled" : "Hidden";
  }
});
```
