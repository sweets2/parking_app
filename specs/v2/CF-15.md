# CF-15 — Static Data Files

## What this is

Two static JSON data files bundled with the app: municipal parking garages and snow emergency routes. These are hand-curated, not scraped — they are committed to the repo.

## Output files

- `data/garages.json`
- `data/snow-emergency-routes.json`

## `data/garages.json`

An array of `Garage` objects (type from `shared/types.ts`):

```typescript
interface Garage {
  name: string;
  lat: number;
  lng: number;
  address: string;
  url?: string;
}
```

Must include all Hoboken municipal parking garages. At minimum:
- Southwest Garage (14th St & Grand St area)
- 2nd Street Garage
- 4th Street Garage (near Observer Hwy)
- Newark St Garage
- Hudson St Garage

Each entry must have accurate Hoboken coordinates (lat ~40.74–40.75, lng ~-74.03 to -74.02).

## `data/snow-emergency-routes.json`

An array of `SnowRoute` objects (type from `shared/types.ts`):

```typescript
interface SnowRoute {
  name: string;
  streets: string[];       // geometry keys matching road-geometry.json keys
  color: string;           // hex color for rendering
  minLon?: number;         // optional east-west clip (for numbered streets)
  maxLon?: number;
  minLat?: number;         // optional north-south clip (for named avenues)
  maxLat?: number;
}
```

Must cover Hoboken's designated snow emergency routes. The `streets` values must match the normalized road geometry keys (e.g., `"WASHINGTON ST"`, `"HUDSON ST"`, `"OBSERVER HWY"`).

### Known constraints on snow route data

1. **Centroid-lat filtering** — `renderSnowEmergencyRoutes` in `app/map.ts` filters out ways whose centroid falls outside `HOBOKEN_STREET_LAT[street] ± HOBOKEN_STREET_LAT_TOLERANCE`. Snow route data for numbered streets must use the correct expected latitude values:
   - `"9TH ST"`: expected lat 40.748 (not 40.750)
   - `"13TH ST"`: expected lat 40.753 (not 40.756)

2. **Per-route clipping** — Use `minLon`/`maxLon` for east-west streets (numbered streets — clip to the cross streets' longitude). Use `minLat`/`maxLat` for north-south streets (e.g., `WILLOW AVE` uses `maxLat: 40.752` to stop at 11th St).

3. **Build script cp block** — Both `data/garages.json` and `data/snow-emergency-routes.json` are already included in the `cp` block of the `build` script in `package.json` (CF-00). Any new static file added to `data/` for this feature must also be added to that cp block.

## Hard constraints

- These are static JSON files — no TypeScript, no tests.
- Coordinates must be real Hoboken coordinates (not placeholders).
- `streets` array values must match the normalized geometry key format used by `normalizeToGeometryKey` in `app/map.ts` (uppercase, abbreviated, no periods).
- Both files must be valid JSON.
