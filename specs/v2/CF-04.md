# CF-04 — Core Parking Logic

## What this is

A pure logic module. No I/O, no browser globals, no side effects. Every time-sensitive function accepts `now: Date` as a parameter — never calls `new Date()` internally.

## Output files

- `shared/parking-logic.ts`
- `tests/unit/parking-logic.test.ts`

### Cross-street and segment helpers

  start: Date;
  end: Date;
  sign: Sign;

export function nextViolationWindow(
  signs: Sign[],

  lng: number,
  now: Date,

export function extractCrossStreets(sign: Sign): [string, string] | null

export function detectMatchingSegment(
  address: string,

  crossStreetCoords: Record<string, { lat: number; lng: number }>,

## Test list (`tests/unit/parking-logic.test.ts`)

Use `NOW_STABLE`, `NOW_AFTER_EXPIRED`, `FETCH_TIME` from `tests/fixtures/signs.ts`. Never use `new Date()` or `Date.now()` in tests.

Tests must cover:

1. `HOBOKEN_BOUNDS` has correct lat/lng min/max values
2. `haversineMeters` returns 0 for identical coords, correct distance for known points
3. `isSignActive` returns true for SIGN_ACTIVE at NOW_STABLE, false at NOW_AFTER_EXPIRED
4. `filterLoadTimeNoise` removes recently-expired signs, keeps active signs
5. `filterActive` returns only signs active at NOW_STABLE
6. `filterNearby` returns signs within radius, excludes those outside
7. `formatCountdown` returns "Expired" for expired sign, countdown string for active sign
8. `formatSignWindow` returns human-readable window string
9. `signSeverity` returns correct severity for tow, cleaning, and other reasons
10. `nudgeCoords` separates signs at identical coords
11. `formatTime` formats a Date as "H:MM AM/PM"
12. `nextViolationWindow` returns nearest violation within 150m
13. `isDataStale` returns false at 24h elapsed, true at 26h elapsed
14. `extractCrossStreets` parses between/and address pattern, returns null for others
15. `detectMatchingSegment` returns correct segment for a known address
16. `getStreetOrientation` returns "EW" for east-west streets, "NS" for north-south

## Hard constraints

- Every time-sensitive function accepts `now: Date`. No internal `new Date()` calls.
- Only allowed import: `type { Sign, RoadGeometry } from "../shared/types"`.
- Tests never call `new Date()` or `Date.now()` — use fixture constants.
