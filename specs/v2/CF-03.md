# CF-03 — Test Fixtures

## What this is

Provides stable test fixtures derived from real fetched data. All time-sensitive tests import constants from this file rather than calling `new Date()`.

## Output files

- `tests/fixtures/signs.ts`

## Module contract

The file reads `data/latest.json` at import time (via `fs.readFileSync`) and exports typed subsets of that data alongside stable date constants.

### Date constants

```typescript
export const FETCH_TIME = new Date("2026-06-09T13:52:50.509Z");
// The fetched_at timestamp from data/latest.json — used as the reference fetch time.

export const NOW_STABLE = new Date("2026-06-09T16:00:00.000Z");
// A stable "now" that is a Monday at noon Eastern Time (2026-06-09T12:00:00-04:00).
// Use this for all time-sensitive test assertions.

export const NOW_AFTER_EXPIRED = new Date("2031-01-01T12:00:00.000Z");
// A "now" well past all real sign expiry dates.
```

### Sign array exports

```typescript
export const ALL_SIGNS: Sign[];       // all signs from data/latest.json
export const ACTIVE_SIGNS: Sign[];    // signs active at FETCH_TIME
export const EXPIRED_SIGNS: Sign[];   // signs not active at FETCH_TIME
export const PERMANENT_SIGNS: Sign[]; // signs with end_iso far in the future (e.g. year 2030+)
```

### Individual sign exports

```typescript
export const SIGN_BAD_COORD: Sign;         // id "215864" — has a coordinate outside Hoboken bounds
export const SIGN_PERMANENT: Sign;         // id "200471" — a permanent permit sign
export const SIGN_ACTIVE: Sign;            // id "213435" — active at FETCH_TIME
export const SIGN_SAME_COORD_A: Sign;      // id "200484" — same lat/lng as SIGN_SAME_COORD_B
export const SIGN_SAME_COORD_B: Sign;      // id "200485" — same lat/lng as SIGN_SAME_COORD_A

// Aliases
export const SIGN_PERMANENT_1: Sign;       // same as SIGN_PERMANENT
export const SIGN_11TH_ST: Sign;           // id "216275" — a sign on 11th St
export const SIGN_EXPIRED_1: Sign;         // first expired sign from EXPIRED_SIGNS
export const SIGN_13TH_ST_A: Sign;         // a sign on 13th St (side A)
export const SIGN_13TH_ST_B: Sign;         // a sign on 13th St (side B)
```

## Hard constraints

- Never call `new Date()` or `Date.now()` in this file. All dates are literal ISO strings passed to `new Date(...)`.
- The file uses `fs.readFileSync` (Node) — no `fetch`, no browser globals.
- All exports are named exports.
- No `as any`. No `!` non-null assertions.
