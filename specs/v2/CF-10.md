# CF-10 — Map Module

## What this is

The ONLY file in the project that may touch Leaflet (`L.*`). All other modules must not import Leaflet. Handles map initialization, all map layer rendering, and street popup display.

## Output files

- `app/map.ts`
- `tests/unit/map.test.ts`

## Private helpers (non-exported)

- `normalizeToGeometryKey(name: string): string` — uppercases, strips periods, contracts suffix abbreviations (STREET→ST, etc.), converts ordinals FIRST→1ST through SIXTEENTH→16TH.
- `getSnappedPinPosition(sign: Sign): { lat: number; lng: number }` — projects sign onto nearest road segment, applies lateral curb offset using `_streetParity` and `PIN_LATERAL_OFFSET_M`. Tracks `bestA`/`bestB` endpoints alongside `projPt`.
- `mergeWays(ways: [number, number][][]): [number, number][][]` — merges end-to-end connected ways into fewer polylines.
- `clipWaysToBounds(ways, minLat, maxLat): [number, number][][]` — latitude clip.
- `clipWaysByBounds(ways, route: SnowRoute): [number, number][][]` — applies per-route minLon/maxLon/minLat/maxLat clip for snow routes.
- `parseLocationLatBounds(location: string, streetKey: string): { minLat: number; maxLat: number } | null` — parses "Xth St. to Yth St." location strings to lat bounds for violation highlight clipping.
- `drawWaysHighlight(ways, color, opacity, side): void` — creates polylines from ways, pushes to `_violationLayers`.
- `drawStreetHighlight(street, color, opacity, side): void` — thin wrapper over `drawWaysHighlight`.

## Known constraints

The following constraints were established during development and must be preserved exactly:

1. **`renderViolationHighlights` is cleaning-only — no signs parameter.** Adding tow signs flooded the map with ~16 permanent permit signs painting nearly every street red at all times. The signature is `renderViolationHighlights(cleaningEntries, now)` with no signs argument.

2. **`normalizeToGeometryKey` must strip periods AND convert ordinals.** Street cleaning data uses abbreviated names with trailing periods ("Adams St.", "Eighth St.") while road-geometry keys use no periods and numeric ordinals ("ADAMS ST", "8TH ST"). The function must: (1) call `.replace(/\./g, "")` before abbreviation expansion, and (2) replace spelled-out ordinals FIRST–SIXTEENTH with 1ST–16TH. Omitting either step causes nearly all cleaning streets to silently produce no highlight.

3. **Snow route centroid-lat filtering.** The OSM bounding box includes Jersey City Heights, so generic street names like "3RD ST" pick up Jersey City ways at lat ~40.753. `renderSnowEmergencyRoutes` guards against this with `HOBOKEN_STREET_LAT` and `HOBOKEN_STREET_LAT_TOLERANCE`. Do not remove this guard.

4. **`data/street-parity.json` must be awaited before `createApp()`.** `renderTowSegments` and `renderSignPins` are called during initial render triggered by `createApp`. `_streetParity` must already be populated at that point. (Note: this constraint applies to main.ts/CF-11, but map.ts must be designed to support it.)

5. **Tow toggle uses explicit layer management, not CSS pane hiding.** `setTowSignsVisible` controls `_signLayers` and `_segmentLayers` via `layer.addTo(_map)` / `layer.remove()`. CSS pane targeting was tried and abandoned as unreliable. `renderSignPins` gates `marker.addTo(_map)` behind `if (_towSignsVisible)`.

6. **Never run `npm run build` during implementation.** The build emits `app/app.js` and `app/sw.js` which shadow TypeScript sources for vitest module resolution. Tests that import `app/app` would then receive the IIFE browser bundle and fail with `TypeError: createApp is not a function`.

7. **`getSnappedPinPosition` applies curb offset.** It tracks `bestA` and `bestB` (endpoints of the nearest road segment) alongside `projPt` and `bestDist`. After the loop it computes `forcedDir` from `_streetParity` and the sign's house number parity, then applies a lateral offset to `projPt` using the right-perpendicular of the best segment (`perpX = dY/len, perpY = -dX/len`).

8. **Per-route coordinate clipping (`minLon`/`maxLon`/`minLat`/`maxLat`).** `SnowRoute` carries optional clip fields. `clipWaysByBounds()` applies these after the centroid-lat filter and before `mergeWays()`. Use `minLon`/`maxLon` for east-west streets (numbered streets — cross streets run north-south at constant longitude) and `minLat`/`maxLat` for north-south streets.

9. **Static data files must be added to the build script's cp block.** Any new static JSON file in `data/` must also have a `cp` step in the `build` script in `package.json`. Omitting this causes the fire-and-forget fetch to 404 silently — no console error, no test failure.

## Hard constraints

- `app/map.ts` is the ONLY file that may touch `L.*` (Leaflet). No other module may import Leaflet.
- No `as any`. No `!`. Named exports only.
- `strict: true` — all types must be explicit.
- Tests mock the Leaflet `L` global — no real Leaflet in tests.
