# CF-07 — Build-Time Geocoding and Road Geometry

## What this is

A collection of build-time scripts (run once, not in the browser) and the runtime geocoding module used by the app:

1. `app/geo.ts` — runtime Nominatim geocoding (used in the browser at click time)
2. `fetcher/geocode-cross-streets.ts` — build-time cross-street coordinate cache
3. `fetcher/build-road-geometry.ts` — Overpass API query to build road geometry JSON
4. `fetcher/build-street-parity.ts` — Overpass API query to build address parity JSON
5. `fetcher/fetch-bus-stops.ts` — NJ Transit ArcGIS query for bus stop locations

## Output files

- `app/geo.ts`
- `fetcher/geocode-cross-streets.ts`
- `fetcher/build-road-geometry.ts`
- `fetcher/build-street-parity.ts`
- `fetcher/fetch-bus-stops.ts`
- `tests/unit/geo.test.ts`

## `app/geo.ts`

Runtime module for reverse geocoding sign addresses from coordinates.

### Exports

```typescript
export function seedGeocodeCache(table: Record<string, { lat: number; lng: number }>): void
// Pre-populates the in-memory intersection cache from a precomputed lookup table.
// Called by main.ts after fetching data/cross-streets.json.

export function getStreetName(lat: number, lng: number): Promise<string | null>
// Reverse geocodes (lat, lng) to a street name via Nominatim.
// Rate-limited to 1 request/sec. Times out after 8 seconds.
// Returns null on error or timeout. Caches results (including null).

export function geocodeCrossStreet(
  streetName: string,
  mainStreet?: string,
): Promise<{ lat: number; lng: number } | null>
// Geocodes a cross-street intersection.
// If mainStreet is provided, uses intersection cache key "mainStreet|streetName".
// If intersection key is not in cache, falls back to standalone streetName lookup.
// Returns null if not found.
```

### Known gotcha — Nominatim User-Agent in Node

Node's built-in `fetch` sends no User-Agent and receives HTTP 403 from Nominatim. The browser sends a User-Agent automatically. Build-time scripts that call Nominatim in Node MUST set an explicit header:

```
"User-Agent": "hoboken-parking-app/1.0 (build-time geocoder)"
```

The runtime `geo.ts` does NOT need this header (the browser provides one automatically).

## `fetcher/geocode-cross-streets.ts`

Build-time script that pre-geocodes all sign cross-street intersections.

### Export

```typescript
export function runGeocode(): Promise<void>
// Reads data/latest.json, extracts all unique cross-street pairs,
// geocodes each (with Nominatim User-Agent header), and writes
// data/cross-streets.json as Record<string, { lat: number; lng: number }>.
```

### Static coordinates

Some boundary terms (e.g. "HUDSON RIVER", "STEVENS PARK") are pre-defined as `STATIC_COORDS` and skipped during geocoding.

### Synthesis strategy

- Phase 1: geocode each unique street name standalone.
- Phase 2: for each intersection, synthesize coordinates:
  - E-W main street: `{ lat: mainCoord.lat, lng: crossCoord.lng }`
  - N-S main street: `{ lat: crossCoord.lat, lng: mainCoord.lng }`
- Cache key format: `"MAIN ST|CROSS ST"` (both uppercase normalized).

## `fetcher/build-road-geometry.ts`

Build-time script that queries Overpass for all road ways in Hoboken.

### Export

```typescript
export function runBuildRoadGeometry(): Promise<void>
// Queries Overpass API for all highway ways within bbox (40.728,-74.060,40.760,-74.022).
// Writes data/road-geometry.json as RoadGeometry (Record<string, [number,number][][]>).
```

### Name normalization (`normalizeName` private function)

Maps full street names to abbreviated keys matching the road geometry lookup used in map.ts:
- STREET → ST, AVENUE → AVE, BOULEVARD → BLVD, HIGHWAY → HWY, PLACE → PL, COURT → CT, DRIVE → DR, ROAD → RD, TERRACE → TER
- Directional abbreviations: NORTH → N, SOUTH → S, EAST → E, WEST → W

## `tests/unit/geo.test.ts`

Tests mock `fetch` with `vi.fn()` and never make real network calls.

1. `getStreetName` returns null on fetch error
2. `getStreetName` returns null on timeout (simulate with delayed resolve)
3. `getStreetName` caches result — second call does not call fetch again
4. `getStreetName` caches null — error result is not retried
5. `geocodeCrossStreet` uses intersection cache key "mainStreet|streetName" when mainStreet is provided
6. `geocodeCrossStreet` falls back to standalone key when intersection key is absent
7. `seedGeocodeCache` pre-populates cache so `geocodeCrossStreet` returns without fetch
8. `getStreetName` Nominatim query uses the correct URL format

## Hard constraints

- No `as any`. No `!`. Named exports only.
- Build-time fetcher scripts must set `User-Agent: "hoboken-parking-app/1.0 (build-time geocoder)"` on Nominatim requests.
- `geo.ts` is a runtime module — it may import types from `shared/types.ts` but must not import Leaflet or any map module.
- Tests mock fetch — no real network calls.
- `normalizeStreet` in `build-street-parity.ts` must be identical in behavior to `normalizeToGeometryKey` in `app/map.ts`.
