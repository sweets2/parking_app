# CF-11 — App Entry Point

## What this is

The browser entry point. Wires the state machine, map, UI, and data fetching together. Built by esbuild from `app/main.ts` into `app/app.js`.

## Output files

- `app/main.ts`
- `tests/unit/main.test.ts`
- `tests/integration/pipeline.test.ts`

### Initialization sequence (inside `initBrowserApp`)

1. `renderLoading()`
2. `initMap()`
3. `initFeedback()`
4. `initCoffee()`
5. Fire-and-forget fetch of `data/street-cleaning.json` (populates `cleaningEntries`) — does NOT block startup
6. Fire-and-forget fetch of `data/cross-streets.json` → `seedGeocodeCache(table)` — does NOT block startup
7. **`await Promise.all([fetch("data/road-geometry.json"), fetch("data/street-parity.json")])`** — both MUST be awaited before `createApp()` runs
8. `await fetch("data/latest.json")` → `allSigns`
9. `await fetch("data/future.json")` → `futureSigns`
10. If a saved spot exists: re-fetch `data/latest.json` with `cache: "no-store"` to get fresh data
11. `correctSignPositions(signs, roadGeometry)`
12. `createApp(deps, initialData)` — state machine starts here
13. Wire toggle buttons: tow, violation, upcoming, garage, snow, bus stop
14. Wire #here-btn and #clear-btn
15. Wire locate button (#locate-btn)
16. `registerMapClickHandler` with the street popup pipeline
17. `wireQueryBar()` — wires query input + mic + submit + clear
18. `setInterval(tick, 60_000)` — 60-second update loop
19. Register service worker (`navigator.serviceWorker.register("sw.js")`)

### Violation refresh scheduling

`scheduleViolationRefresh()` schedules a call to `renderViolationHighlights(cleaningEntries, now)` at the next :00 hour boundary using `setTimeout`. Each call reschedules the next one.

## `tests/unit/main.test.ts` — test cases

Mock: `app/map`, `app/ui`, `shared/storage`, `app/app` with `vi.mock`.

1. `normalizeStreet("Washington St.")` → `"washington street"`
2. `normalizeStreet("1st Ave")` → `"1st avenue"`
3. `normalizeStreet("Seventh St")` → `"7th street"`
4. `findCleaningEntries` returns entries matching the normalized road name
5. `findCleaningEntries` returns empty array when no entries match
6. `setQueryActiveLabel` sets #query-bar hidden=true, #query-active-bar hidden=false
7. `setQueryIdle` sets #query-active-bar hidden=true, #query-bar hidden=false
8. Query input: pressing Enter calls onQuerySubmit with trimmed text
9. Query input: empty text → onQuerySubmit NOT called
10. Query clear button: calls onQueryClear, returns to idle state
11. `initBrowserApp` awaits road-geometry and street-parity before createApp

## Hard constraints

- No `as any`. No `!`. Named exports only.
- `road-geometry.json` and `street-parity.json` MUST be in `Promise.all` — not fire-and-forget.
- `street-cleaning.json` and `cross-streets.json` are fire-and-forget after step 2.
- esbuild bundles from `app/main.ts`, not `app/app.ts`.
- Never run `npm run build` during implementation.
- Import `parseQuery` from `../shared/query-parser` and `renderSweepResults`/`clearSweepResults` from `./map`.
