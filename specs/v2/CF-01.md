# CF-01 — CI/CD and GitHub Actions

## What this is

A GitHub Actions workflow that runs the data fetcher on a daily schedule and commits the result.

## Output files

- `.github/workflows/fetch_daily.yml`

## Workflow specification

- **Trigger:** `schedule` cron `0 14 * * *` (2 pm UTC daily) and `workflow_dispatch` (manual trigger).
- **Job:** `fetch` running on `ubuntu-latest`.
- **Steps:**
  1. Checkout repo (with `persist-credentials: true` so the push step works).
  2. Set up Node.js (version 20).
  3. Install dependencies: `npm install`.
  4. Run fetcher: `npm run fetch`.
  5. Commit and push if data changed:
     ```bash
     git config user.name "github-actions[bot]"
     git config user.email "github-actions[bot]@users.noreply.github.com"
     git add data/latest.json data/future.json
     git diff --staged --quiet || git commit -m "chore: daily parking data update"
     git push
     ```
- **Environment variable:** `API_URL` should be available via `secrets.API_URL` or set to the default Hoboken API base URL.

## Hard constraints

- No TypeScript. No tests. Output is YAML only.
- The workflow file must be valid YAML.
- Do not commit `data/street-cleaning.json` or other build artifacts — only `data/latest.json` and `data/future.json`.
- TypeScript `strict: true` always on. No `as any` casts. No `!` non-null assertions. All exported symbols use named exports — no default exports.
- Never run `npm run build` during implementation (build emits app.js/sw.js which shadow TypeScript sources in vitest).
- Tests run in Node (`environment: "node"` in vitest.config). No browser globals available at test time.
