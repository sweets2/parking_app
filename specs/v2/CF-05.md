# CF-05 — Storage Module

## What this is

Manages persistence of the user's saved parking spot using an injectable storage backend. All localStorage access is inside the factory object — never at module scope.

## Output files

- `shared/storage.ts`
- `tests/unit/storage.test.ts`

## `shared/storage.ts` — exports

### `StorageBackend`

```typescript
export interface StorageBackend {
  getItem(key: string): string | null;
  setItem(key: string, value: string): void;
  removeItem(key: string): void;
}
```

### `SavedSpot`

```typescript
export interface SavedSpot {
  lat: number;
  lng: number;
  side?: string;
  savedAt: string;  // ISO timestamp string
  address: string;
}
```

### `SpotStorage`

```typescript
export interface SpotStorage {
  save(spot: SavedSpot): void;
  load(): SavedSpot | null;
  clear(): void;
}
```

### `createSpotStorage`

```typescript
export function createSpotStorage(backend: StorageBackend): SpotStorage
```

- Uses key `"hoboken_parking_spot"` (module-level constant `STORAGE_KEY`).
- `save`: serializes spot to JSON and calls `backend.setItem`.
- `load`: calls `backend.getItem`, parses JSON, validates shape, returns `SavedSpot | null`.
- `clear`: calls `backend.removeItem`.
- If `backend.getItem` returns null or invalid JSON, `load` returns null (no throw).
- `backend` is never called at module scope — only inside the returned factory methods.

## Test list (`tests/unit/storage.test.ts`)

Tests use a Map-backed fake backend:

```typescript
function makeFakeBackend(): StorageBackend {
  const store = new Map<string, string>();
  return {
    getItem: (k) => store.get(k) ?? null,
    setItem: (k, v) => { store.set(k, v); },
    removeItem: (k) => { store.delete(k); },
  };
}
```

### Test cases

**Test 1 — factory returns correct interface**

Given `createSpotStorage` is called with a fake backend,
When the return value is inspected,
Then it has properties `save`, `load`, and `clear` that are all functions.

**Test 2 — load returns null on empty storage**

Given a fresh `SpotStorage` backed by an empty fake backend,
When `load()` is called,
Then the result is `null`.

**Test 3 — save then load round-trips spot data**

Given a `SpotStorage` backed by a fake backend,
When `save({ lat: 40.744, lng: -74.032, savedAt: "2024-01-01T10:00:00.000Z", address: "1st St" })` is called and then `load()` is called,
Then the returned object has `lat` equal to `40.744`, `lng` equal to `-74.032`, `savedAt` equal to `"2024-01-01T10:00:00.000Z"`, and `address` equal to `"1st St"`.

**Test 4 — clear removes saved spot**

Given a `SpotStorage` backed by a fake backend with a spot already saved,
When `clear()` is called and then `load()` is called,
Then the result is `null`.

**Test 5 — load returns null for invalid JSON**

Given a `SpotStorage` whose backend returns the string `"not-valid-json"` for key `"hoboken_parking_spot"`,
When `load()` is called,
Then the result is `null` and no exception is thrown.

**Test 6 — load returns null for wrong shape**

Given a `SpotStorage` whose backend returns `'{"foo":"bar"}'` for key `"hoboken_parking_spot"`,
When `load()` is called,
Then the result is `null`.

## Hard constraints

- Must NOT call `localStorage` at module scope.
- `createSpotStorage` takes a `StorageBackend` parameter — no direct `localStorage` access inside the module.
- Tests use `environment: "node"` — no browser globals. Use the Map-backed fake.
- No `as any`. No `!`. Named exports only.
