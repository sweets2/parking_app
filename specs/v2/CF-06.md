# CF-06 — Data Fetchers

## What this is

Two fetcher scripts and their tests:

1. `fetcher/fetch.ts` — fetches live tow-zone sign data from the Hoboken parking API.
2. `fetcher/fetch-street-cleaning.ts` — scrapes street cleaning schedule from the Hoboken city website.

## Output files

- `fetcher/fetch.ts`
- `fetcher/fetch-street-cleaning.ts`
- `tests/unit/fetcher.test.ts`

---

### Data written

`data/latest.json`:
```json
{
  "fetched_at": "<ISO timestamp>",
  "count": <number>,
  "signs": [ ... ]
}
```

Each sign in `signs` is a normalized `Sign` object (types from `shared/types.ts`):
- `id`: string
- `address`: string
- `reason`: string
- `permit_number`: string
- `lat`: number (parsed from API string)
- `lng`: number (parsed from API string)
- `start_date`, `start_time`, `stop_date`, `end_time`: raw strings
- `start_iso`, `end_iso`: computed by `toIsoDatetime`
- `active_at_fetch`: computed by `computeActiveAtFetch`

## `fetcher/fetch-street-cleaning.ts`

Scrapes `https://www.hobokennj.gov/resources/street-cleaning-schedule`.

### Exports

```typescript

// - Replaces dash-separated day ranges with "through" (e.g. "Monday-Friday" → "Monday through Friday")
// - Inserts a triple-space separator between the day/range part and the time part
//   (e.g. "Monday through Friday   8 am – 9 am")
// - Normalizes "AM"/"PM" to lowercase "am"/"pm"

// Parses the Webflow CMS HTML and returns all entries.
// Uses div.w-dyn-item selector; never uses table/tr selectors.
// Skips any entry where street === "Street" (the header row).

export function runScraper(): Promise<void>
// Fetches https://www.hobokennj.gov/resources/street-cleaning-schedule,
// calls parseCleaningHtml on the response text,
// and writes the resulting array to data/street-cleaning.json as JSON.
```

### Known page layout gotcha

The page is built with Webflow CMS and uses `div` elements, NOT `<table>`. Do not use `table`, `tbody`, or `tr` selectors — they will match nothing.

- Each row is a `div.w-dyn-item`.
- Inside each item, skip the `div.table_wrapper` that has class `w-condition-invisible` (a hidden mobile duplicate); use the other one.
- The four `div.table-content` children hold: Street, Side, Days & Hours, Location — in that order.
- The column headers appear as a real `div.w-dyn-item` entry (not a `<thead>`). Filter it out: skip any parsed entry where `street === "Street"`.

### Schedule normalization

`normalizeSchedule` must:
- Replace "Monday-Friday" with "Monday through Friday" (and similar dash-separated day ranges)
- Insert a triple-space (`   `) between the day/range part and the time part (e.g. `"Monday through Friday   8 am – 9 am"`)
- Normalize "AM"/"PM" to lowercase "am"/"pm"

### Test list (`tests/unit/fetcher.test.ts`)

1. `toIsoDatetime("6/9/2026", "08:00:00")` returns `"2026-06-09T08:00:00"`
2. `validateResponseShape` returns false for non-object (e.g. `null`), true for valid shape (`{ data: [] }`)
3. `validateSign` returns false for missing required fields (e.g. `{}`), true for a valid sign object with all required string fields
4. `checkCountDrop` does not throw when counts are similar (e.g. `checkCountDrop(100, 100)` returns `undefined`)
5. `normalizeSchedule("Monday-Friday 8 AM – 9 AM")` returns a string containing `"Monday through Friday"` (dash-separated day ranges replaced with "through")
6. `normalizeSchedule("Monday through Friday 8 AM – 9 AM")` returns a string containing `"   "` (triple-space separator inserted between day part and time part)
7. `normalizeSchedule("Monday-Friday 8 AM – 9 AM")` returns a string where `"AM"` has been lowercased to `"am"` (am/pm capitalization normalized)
8. `parseCleaningHtml("")` returns `[]` (empty array for empty HTML)
9. `parseCleaningHtml` skips the header row: given HTML with a single `div.w-dyn-item` whose first `div.table-content` text is `"Street"`, returns `[]`
10. `parseCleaningHtml` correctly parses a well-formed Webflow item div: given HTML with one valid `div.w-dyn-item` containing four `div.table-content` children with texts `["Adams St", "North", "Monday-Friday 8 AM – 9 AM", "Full Block"]`, returns an array of length 1 with `street === "Adams St"`, `side === "North"`, and `location === "Full Block"`
11. `computeActiveAtFetch` returns `true` when `fetchedAt` is within the sign interval: given a sign with `start_iso: "2026-06-09T13:52:50"` and `end_iso: "2026-11-30T23:59:59"` and `fetchedAt: FETCH_TIME.toISOString()`, returns `true`
12. `computeActiveAtFetch` returns `false` when `fetchedAt` is outside the sign interval: given a sign with `start_iso: "2026-06-01T00:00:00"` and `end_iso: "2026-06-08T23:59:59"` and `fetchedAt: FETCH_TIME.toISOString()` (which is `"2026-06-09T13:52:50.509Z"`, after end_iso), returns `false`
13. `runFetcher` writes `data/latest.json` via the FsBackend when the API returns a valid response: given a mock `fetch` that resolves with `{ data: [<one valid RawSign>] }` and a spy FsBackend, `writeFileSync` is called with a path containing `"latest.json"` and a JSON string containing `"fetched_at"`
14. `runFutureFetcherWithFs` writes `data/future.json` via the FsBackend when the API returns a valid response: given a mock `fetch` that resolves with `{ data: [] }` and a spy FsBackend, `writeFileSync` is called with a path containing `"future.json"`
15. `runFetcherWithFs` calls `runFetcher` before `runFutureFetcherWithFs`: given a recording FsBackend and mock fetch, the call order of writes reflects main endpoint before future endpoint
16. `runScraper` fetches the street cleaning URL and writes `data/street-cleaning.json`: given a mock `fetch` that returns valid Webflow HTML, `writeFileSync` is called with a path containing `"street-cleaning.json"`

## Hard constraints

- No `as any`. No `!`. Named exports only.
- `parseCleaningHtml` must use `div.w-dyn-item` selector, not table/tr selectors.
- All fetcher scripts are Node-only; no browser globals in tests.
- Tests run in `environment: "node"`.
- `runFetcherWithFs` runs `runFetcher` then `runFutureFetcherWithFs` sequentially (not with `Promise.all`). If `runFetcher` rejects, `runFutureFetcherWithFs` is not called.
- Tests 11 and 12 must use `FETCH_TIME` from `tests/fixtures/signs.ts` for the `fetchedAt` argument (via `FETCH_TIME.toISOString()`). Do not use `new Date()` or a literal date string in the test body.
