# CF-08 — App State Machine

## What this is

A pure state machine module for the app. No DOM access, no side effects, no setInterval. All state transitions happen through explicit method calls.

## Output files

- `app/app.ts`
- `tests/unit/app.test.ts`

## `app/app.ts` — exports

### Types

```typescript
export type AppState = "loading" | "error" | "browsing" | "parked";
```

### `App` interface

```typescript
export interface App {
  getState(): AppState;
  setUserPosition(lat: number, lng: number): void;
  onSaveSpot(lat: number, lng: number, address: string): void;
  onClearSpot(): void;
  onHereNow(): void;
  tick(now: Date): void;
}
```

### `AppDeps` interface (passed to `createApp`)

```typescript
export interface AppDeps {
  storage: SpotStorage;
  onStateChange(state: AppState): void;
  onSigns(signs: Sign[]): void;
  onNearby(nearby: Sign[]): void;
  onStale(hoursAgo: number | null): void;
}
```

### `createApp`

```typescript
export function createApp(deps: AppDeps, initialData: ParkingData): App
```

- Initial state: `"parked"` if `deps.storage.load()` returns a non-null spot; otherwise `"browsing"`.
- `onSaveSpot`: saves spot to storage, transitions to `"parked"`, calls `deps.onStateChange`.
- `onClearSpot`: clears storage, transitions to `"browsing"`, calls `deps.onStateChange`.
- `tick(now)`: re-evaluates nearby signs and staleness, calls `deps.onNearby` and `deps.onStale` as appropriate.
- No `setInterval`, no `document`, no `window`, no `fetch`.

## Test list (`tests/unit/app.test.ts`)

Tests use a Map-backed fake storage backend (same pattern as CF-05 storage tests).

1. Initial state is `"browsing"` when no saved spot exists
2. Initial state is `"parked"` when a saved spot exists in storage
3. `onSaveSpot` transitions from browsing to parked
4. `onClearSpot` transitions from parked to browsing
5. `onSaveSpot` calls `deps.storage.save` with the correct spot data
6. `onClearSpot` calls `deps.storage.clear`
7. `tick` calls `deps.onNearby` with signs near the user position
8. `tick` calls `deps.onStale` with null when data is fresh
9. `tick` calls `deps.onStale` with hours elapsed when data is stale (> 25 hours)
10. `onStateChange` is called on every state transition

## Hard constraints

- No `as any`. No `!`. Named exports only.
- No DOM access, no `window`, no `document`, no `setInterval`, no `fetch`.
- `app/app.ts` exports state machine only — no side effects at module scope.
- Tests use `environment: "node"` — no browser globals.
