# CF-09 — UI Rendering Module

## What this is

DOM rendering functions for the app UI. All functions accept explicit parameters — no module-level state that holds DOM references.

## Output files

- `app/ui.ts`
- `tests/unit/ui.test.ts`

## `app/ui.ts` — exports

### Constants

```typescript
export const TOAST_DURATION_MS = 4000;
```

### Functions

```typescript
export function renderLoading(): void
// Shows a loading indicator in the DOM.

export function hideLoading(): void
// Removes or hides the loading indicator.

export function renderSignCards(signs: Sign[], now: Date): void
// Renders sign cards into #sign-list (which is display:none — not visible to users,
// but the DOM update still occurs for programmatic access).
// Signs sorted by severity: high → medium → low.

export function renderBrowsingMode(activeSigns: Sign[], now: Date): void
// Updates UI elements for browsing mode (no saved spot).

export function renderClearBanner(): void
// Shows the "you are parked here" banner with a clear button.

export function renderWarningBanner(nearbySigns: Sign[], now: Date): void
// Shows a warning banner when nearby active restrictions exist.

export function showSpotToast(address: string): void
// Shows a temporary toast notification with the saved spot address.
// Auto-dismisses after TOAST_DURATION_MS milliseconds.

export function showStaleBanner(hoursAgo: number): void
// Shows a banner indicating data is stale, with how many hours old it is.
```

## Test list (`tests/unit/ui.test.ts`)

Tests install a minimal fake DOM on `globalThis` before each test:

```typescript
// Minimal fake DOM setup
globalThis.document = {
  getElementById: (id: string) => fakeElements[id] ?? null,
  createElement: (tag: string) => ({ tagName: tag, children: [], style: {}, classList: { add(){}, remove(){}, contains(){ return false; } }, textContent: "", innerHTML: "", hidden: false }),
} as unknown as Document;
```

1. `renderLoading` does not throw when #loading element is absent
2. `hideLoading` does not throw when #loading element is absent
3. `renderBrowsingMode` hides #here-btn and #clear-btn when no signs are active
4. `renderClearBanner` does not throw when banner element is absent
5. `showSpotToast` does not throw when toast container is absent
6. `renderSignCards` sorts signs: high severity first, then medium, then low
7. `showStaleBanner` does not throw when banner element is absent
8. `renderWarningBanner` does not throw when banner element is absent

## Hard constraints

- No `as any`. No `!`. Named exports only.
- `#sign-list` is `display:none` — do not unhide it or reference it as visible UI.
- `#here-btn` and `#clear-btn` are `display:none` initially — ui.ts controls their visibility.
- Tests run in `environment: "node"` — install a fake DOM on globalThis rather than using jsdom.
