# CF-25 — Cross-Street Segment Interpolation and PAD Reduction

## Module
`app/map.ts` (modified), `tests/unit/map.test.ts` (modified)

## Problem
`getCrossStreetCoord` finds a cross-street's coordinate by snapping to the **nearest OSM node** on any matching way. With `PAD = 0.0003` (≈33 m) added to the clip bound, the highlight overshoots the terminus intersection — Park Ave's "Observer Hwy. to Fourteenth St." clean zone extends ~20 m north of 14th St.

Root cause: the nearest node on 14th St near Park Ave's median longitude sits at lat=40.753875. Adding PAD=0.0003 clips at 40.754175, which is ~20 m past the visual 14th St centerline (~40.754). Nearest-node snapping accumulates error proportional to how far the OSM node sits from the exact crossing point.

## Goal
Replace point-snap with **segment interpolation** in `getCrossStreetCoord`, and reduce PAD from 0.0003° to 0.0001° (≈11 m). Together these changes cut the Park Ave overshoot from ~20 m to ~1–4 m and improve accuracy for all streets.

## Changes — `app/map.ts`

### New module-level constant (before `LOCATION_RANGE_RE`)

```typescript
// Small visual buffer past the exact cross-street crossing. With segment interpolation,
// the crossing coordinate is accurate to ~1 m; 0.0001° (≈11 m) keeps the endpoint
// visually connected to the cross street without overshooting.
// TODO: remove when rendering switches to precomputed cleaning-segments.json arcs.
const LOCATION_CLIP_PAD_DEG = 0.0001;
```

### `getCrossStreetCoord` — add segment-crossing pass before nearest-node fallback

For each way, iterate segments first. If a segment straddles `mainCoord` on the query axis, interpolate the exact return coordinate. Set `bestDist = 0` so the nearest-node fallback loop cannot overwrite it.

```typescript
for (const way of ways) {
  // Segment-crossing pass: find exact coord where cross-street crosses mainCoord.
  for (let i = 1; i < way.length; i++) {
    const prev = way[i - 1];
    const curr = way[i];
    if (prev === undefined || curr === undefined) continue;
    const prevQuery = axis === "lat" ? prev[0] : prev[1];
    const currQuery = axis === "lat" ? curr[0] : curr[1];
    if (currQuery === prevQuery) continue;
    if (mainCoord < Math.min(prevQuery, currQuery) || mainCoord > Math.max(prevQuery, currQuery)) continue;
    const t = (mainCoord - prevQuery) / (currQuery - prevQuery);
    const prevRet = axis === "lat" ? prev[1] : prev[0];
    const currRet = axis === "lat" ? curr[1] : curr[0];
    const interp = prevRet + t * (currRet - prevRet);
    if (validRange !== undefined && (interp < validRange[0] || interp > validRange[1])) continue;
    if (0 < bestDist) { bestDist = 0; best = interp; }
    break;
  }
  // Nearest-node fallback: keeps existing behaviour when no segment crossing found.
  // bestDist=0 after a crossing prevents any node from overwriting the interpolated result.
  for (const pt of way) { /* … existing loop … */ }
}
```

### `parseLocationBounds` — replace local `PAD` with `LOCATION_CLIP_PAD_DEG`

Remove `const PAD = 0.0003;` (and its comment block). Replace both `PAD` usages with `LOCATION_CLIP_PAD_DEG`. Keep `RANGE_PAD = 0.003` unchanged (it is a contamination filter, not a visual tolerance).

### New regression test: Park Ave / 14th St overshoot (new `describe("CF-25 ...")` block)

**Given/When/Then:**

**Test 1 — Park Ave does not overshoot 14th St (requires both interpolation AND reduced PAD)**
- Given: `initRoadGeometry` with:
  - `"PARK AVE"`: N-S way `[[40.734, -74.034], [40.748, -74.031], [40.754, -74.029], [40.757, -74.028]]`
  - `"OBSERVER HWY"`: horizontal E-W way `[[40.734, -74.032], [40.734, -74.027]]`
  - `"14TH ST"`: diagonal segment `[[40.7543, -74.031], [40.7537, -74.027]]`
    - Neither node is at mainLng (-74.029); crossing at t=0.5 → interp lat=40.7540
    - Nearest-node + old PAD: clip max = 40.7543+0.0003 = 40.7546 → fails threshold
    - Nearest-node + new PAD: clip max = 40.7543+0.0001 = 40.7544 → still fails threshold
    - Segment interpolation + new PAD: clip max = 40.7540+0.0001 = 40.7541 → passes threshold
- And: one cleaning entry `{ street: "Park Ave.", side: "East", schedule: "Monday   1 pm – 2 pm", location: "Observer Hwy. to Fourteenth St." }`
- When: `renderViolationHighlights([entry], new Date("2026-06-23T17:00:00.000Z"))` (Monday 1 pm EDT = active)
- Then: all rendered polyline lat coordinates ≤ 40.754 + 0.00012 (= 40.75412)
- And: at least one polyline point was rendered

Threshold 40.75412 catches nearest-node+old-PAD (~0.7546), nearest-node+new-PAD (~0.7544), while passing the fix (~0.7541). Both segment interpolation AND PAD reduction are required to pass.
