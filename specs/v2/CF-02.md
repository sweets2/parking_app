# CF-02 — Shared Types

## What this is

Defines every TypeScript type and constant shared across the app. No logic — types only.

## Output files

- `shared/types.ts`

## Exports

### Constants

```typescript
export const SIGN_REASONS = [
  "CONSTRUCTION",
  "SPECIAL EVENT",
  "STREET CLEANING",
  "UTILITY WORK",
  "OTHER",
  // ... all reason strings from the API
] as const;

export type SignReason = (typeof SIGN_REASONS)[number];
```

### `Sign`

```typescript
export interface Sign {
```

### `RawSign` and `RawApiResponse`

Raw shapes exactly as returned by the API (before normalization). Fields may be snake_case strings; types are looser (string | number for lat/lng, etc.).

## Hard constraints

- No logic, no functions, no class declarations.
- All exports are named exports (`export interface`, `export type`, `export const`).
- No `as any`. No `!` non-null assertions.
- `strict: true` must compile without errors.
- TypeScript `strict: true` always on. No `as any` casts. No `!` non-null assertions. All exported symbols use named exports — no default exports.
- Never run `npm run build` during implementation (build emits app.js/sw.js which shadow TypeScript sources in vitest).
- Tests run in Node (`environment: "node"` in vitest.config). No browser globals available at test time.
