## F-40 Coffee Donation Button

Add a floating coffee-cup icon button directly above the existing feedback (mail) button.
Clicking it opens a small popover card with a brief message and a Ko-fi donation link.
The popover dismisses on outside click or Escape. No full-screen overlay — this is
intentionally lighter than the feedback modal.

Files created:
- `tests/unit/coffee.test.ts` — DOM structure and behavior tests

Files modified:
- `app/index.html` — add `#coffee-btn` and `#coffee-popover` elements
- `app/style.css` — style the button (42 px circle, matches `#feedback-btn`) and popover card
- `app/main.ts` — wire click/outside-click/Escape handlers in `initBrowserApp()`

### `app/main.ts` — modifications

Extract coffee wiring into a standalone exported function `initCoffee()` and call it
from `initBrowserApp()` after `initFeedback()`.

```typescript
export function initCoffee(): void {
  const coffeeBtn = document.getElementById('coffee-btn');
  const coffeePopover = document.getElementById('coffee-popover');

  coffeeBtn?.addEventListener('click', (e: MouseEvent) => {
    e.stopPropagation();
    const isOpen = coffeePopover?.classList.contains('open') ?? false;
    coffeePopover?.classList.toggle('open', !isOpen);
    coffeePopover?.setAttribute('aria-hidden', String(isOpen));
  });

  document.addEventListener('click', (e: MouseEvent) => {
    if (coffeePopover?.classList.contains('open') &&
        !coffeePopover.contains(e.target as Node)) {
      coffeePopover.classList.remove('open');
      coffeePopover.setAttribute('aria-hidden', 'true');
    }
  });

  document.addEventListener('keydown', (e: KeyboardEvent) => {
    if (e.key === 'Escape') {
      coffeePopover?.classList.remove('open');
      coffeePopover?.setAttribute('aria-hidden', 'true');
    }
  });
}
```

### `tests/unit/coffee.test.ts` — new file

Test environment: `jsdom` (set via vitest `@vitest-environment jsdom` docblock comment).

Test setup: before each test, create a minimal DOM with `#coffee-btn`, `#coffee-popover`,
`.coffee-msg`, and `#coffee-link` using `document.body.innerHTML`. Import and call
a helper that wires the coffee handlers (or re-use a small extracted helper if the
Creator chooses to extract the wiring into a `initCoffee()` function in `main.ts`).

Because `initBrowserApp()` calls `initMap()` and many other browser-only initialisers,
**do not call `initBrowserApp()` in tests**. Instead, extract the coffee-button wiring
into a standalone exported function `initCoffee()` in `app/main.ts` and call that
from both `initBrowserApp()` (replacing the inline block) and from the tests.

`initCoffee()` signature:

```typescript
export function initCoffee(): void { /* wiring code */ }
```

Test cases:

1. GIVEN the DOM contains `#coffee-btn` and `#coffee-popover`, WHEN `initCoffee()` is called and `#coffee-btn` is clicked, THEN `#coffee-popover` has class `open`.

2. GIVEN `#coffee-popover` already has class `open`, WHEN `#coffee-btn` is clicked again, THEN `#coffee-popover` does not have class `open` (toggle off).

3. GIVEN `#coffee-popover` has class `open`, WHEN a click event is dispatched on `document.body` (outside the popover), THEN `#coffee-popover` does not have class `open`.

4. GIVEN `#coffee-popover` has class `open`, WHEN a `keydown` event with `key: "Escape"` is dispatched on `document`, THEN `#coffee-popover` does not have class `open`.

5. GIVEN `app/index.html` source (read as a string), THEN it contains `id="coffee-btn"`.

6. GIVEN `app/index.html` source (read as a string), THEN it contains `id="coffee-popover"`.

7. GIVEN `app/index.html` source (read as a string), THEN `#coffee-link` has `href="https://ko-fi.com/neighborhooddev"`.

8. GIVEN `app/index.html` source (read as a string), THEN `#coffee-link` has `rel="noopener noreferrer"`.

For tests 5–8, read the file using Node `fs.readFileSync` (tests run in Node/jsdom, `fs`
is available). Do not use `fetch`.

### Known gotchas

**`initCoffee` must be exported** so tests can call it in isolation without invoking
`initBrowserApp()` and its browser-only dependencies (`initMap`, GPS, etc.).

**`aria-hidden` toggle** — when the popover opens, set `aria-hidden="false"`; when it
closes (outside click, Escape, or second button click), set `aria-hidden="true"`. The
initial state in HTML is `aria-hidden="true"`.

**`e.stopPropagation()` is required** on the coffee button click handler. Without it the
click event bubbles to `document` and the outside-click listener immediately closes the
popover that was just opened.
