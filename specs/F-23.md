## F-23 Tow Zone Segment Highlight `[TODO]`

Each active tow/no-parking sign currently appears as a small SVG icon pin. This feature draws a short red polyline (~18 m) on the map for each active sign, visually marking the curb segment where parking is restricted.

**Core insight:** `sign.lat` / `sign.lng` already encodes the correct curb side — the sign is physically posted on the restricted curb. No additional geocoding is needed. Street orientation (E-W or N-S) is derived from the street name in `sign.address`. The polyline is drawn through the sign position along the street axis.

**Hoboken grid rules:**
- Numbered streets ("1ST ST" … "14TH ST") run East-West.
- Named streets ("BLOOMFIELD ST", "WASHINGTON ST") run North-South.
- Two named EW exceptions: OBSERVER HWY, NEWARK ST.

Four files are added or modified:

- `shared/parking-logic.ts` — new `getStreetOrientation` export
- `app/map.ts` — polyline type shim, `_segmentLayers` + `_towSignsVisible` state, `renderTowSegments` export, updated `setTowSignsVisible`, updated `initMap`
- `app/main.ts` — call `renderTowSegments` alongside every `renderSignPins` call
- `tests/unit/parking-logic.test.ts` — tests for `getStreetOrientation`
- `tests/unit/map.test.ts` — polyline mock + tests for `renderTowSegments` and `setTowSignsVisible` with segments

### Module: `shared/parking-logic.ts` — new export

#### `getStreetOrientation(address: string): "EW" | "NS"` (exported named)

Strips the leading house-number prefix from `address` and classifies the street as East-West or North-South.

Algorithm:
1. Strip the leading house-number token: `address.replace(/^\d[\d-]*\s+/, "").trim()`. The regex `/^\d[\d-]*/` matches both single-number (`"257 11TH ST"`) and range (`"257-257 11TH ST"`) formats. Using the range-only pattern `/^\d+-\d+/` is a bug — it leaves house digits in the string and causes false "EW" for NS streets.
2. If the stripped street name contains a digit (`/\d/.test(street)`) → return `"EW"` (numbered streets like "11TH ST").
3. If the stripped street name is in the hardcoded `EW_NAMED` set → return `"EW"`. `EW_NAMED = new Set(["OBSERVER HWY", "NEWARK ST"])`. These streets are E-W but contain no digit.
4. Otherwise → return `"NS"`.

`EW_NAMED` is module-level (not inside the function) so the Set is constructed once.

No new imports needed — this function accepts only `string`.

#### `renderTowSegments(signs: Sign[]): void` (new exported named function)

```typescript
const SEG_HALF_LAT = 0.000162; // ≈18 m north/south
const SEG_HALF_LNG = 0.000215; // ≈18 m east/west at 40.74°N

export function renderTowSegments(signs: Sign[]): void {
  for (const layer of _segmentLayers) {
    layer.remove();
  }
  _segmentLayers = [];

  if (_map === null) return;
  const L = getL();

  for (const sign of signs) {
    const orientation = getStreetOrientation(sign.address);
    const latlngs: [[number, number], [number, number]] =
      orientation === "EW"
        ? [[sign.lat, sign.lng - SEG_HALF_LNG], [sign.lat, sign.lng + SEG_HALF_LNG]]
        : [[sign.lat - SEG_HALF_LAT, sign.lng], [sign.lat + SEG_HALF_LAT, sign.lng]];

    const segment = L.polyline(latlngs, { color: "#cc0000", weight: 5, opacity: 0.75 });
    _segmentLayers.push(segment);
    if (_towSignsVisible) {
      segment.addTo(_map);
    }
  }
}
```

Note: `_segmentLayers` is cleared before the `_map === null` guard so stale references are dropped even when called before `initMap()`.

#### Update `setTowSignsVisible(visible: boolean): void`

The current implementation early-returns when `mapEl === null`, which would skip segment management in the Node test environment (where `document.getElementById` always returns null). The new version removes the early return and separates the two concerns:

```typescript
export function setTowSignsVisible(visible: boolean): void {
  _towSignsVisible = visible;
  const mapEl = document.getElementById("map");
  if (mapEl !== null) {
    if (visible) {
      mapEl.classList.remove("tow-signs-hidden");
    } else {
      mapEl.classList.add("tow-signs-hidden");
    }
  }
  if (_map === null) return;
  for (const layer of _segmentLayers) {
    if (visible) {
      layer.addTo(_map);
    } else {
      layer.remove();
    }
  }
}
```

`_towSignsVisible` is set unconditionally first so the flag is accurate even when `mapEl` is absent.

#### Pair `renderTowSegments` with each `renderSignPins` call

There are four `renderSignPins` call sites. Immediately after each one, call `renderTowSegments` with the same `signs` argument (without the `now` parameter — `renderTowSegments` derives orientation from the address string, not from time):

1. Browsing mode: after `renderSignPins(state.activeSigns, now)` add `renderTowSegments(state.activeSigns)`
2. Parked mode: after `renderSignPins(filterActive(state.allSigns, now), now)` add `renderTowSegments(filterActive(state.allSigns, now))`

4. Browsing silent refresh: after `renderSignPins(activeNow, now)` add `renderTowSegments(activeNow)`

In case 2, `filterActive(state.allSigns, now)` is called twice (once for each render function). This is consistent with the surrounding code style — no temporary variable is introduced.
