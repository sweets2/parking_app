## F-28 Floating Feedback Button `[TODO]`

A circular floating button sits in the bottom-right corner of the map. Clicking it opens an in-app modal where users can type feedback and submit it. Submissions are POSTed to a Vercel serverless function (`/api/feedback`) which forwards them to the developer's inbox via the Resend email API. The developer's email address never appears in client-side code.

### Module: `app/feedback.ts` — new file

#### `initFeedback(): void` (exported named)

Reads the six required DOM elements via `document.getElementById`. If any element is missing (returns `null`), return early without throwing.

```
elements: feedback-btn, feedback-overlay, feedback-cancel,
          feedback-submit (HTMLButtonElement), feedback-text (HTMLTextAreaElement),
          feedback-confirm
```

**Open:** `#feedback-btn` click → add class `open` to overlay, set `aria-hidden="false"` on overlay, call `focus()` on textarea.

**`closeModal()` (internal helper):**
- Remove class `open` from overlay
- Set `aria-hidden="true"` on overlay
- Clear textarea `value`
- Clear `#feedback-confirm` `textContent` and `className`
- Set `#feedback-submit` `disabled = false`

**Cancel:** `#feedback-cancel` click → `closeModal()`.

**Overlay background click:** `#feedback-overlay` click where `event.target === overlay` → `closeModal()`. Clicks on the inner modal element must NOT close the overlay.

**Submit:**
1. Read trimmed value from `#feedback-text`; if empty, return without calling `fetch`
2. Set `#feedback-submit` `disabled = true`
3. `fetch("/api/feedback", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ message: trimmedValue }) })`
4. On resolved `response.ok === true`: set `#feedback-confirm` textContent `"Thanks — feedback received!"`, className `"success"`; schedule `closeModal()` via `setTimeout` after 2200 ms
5. On resolved `response.ok === false`: set textContent `"Something went wrong. Please try again."`, className `"error"`, re-enable submit
6. On fetch rejection: set textContent `"Could not send. Check your connection."`, className `"error"`, re-enable submit

### Module: `api/feedback.ts` — new file

Vercel Edge Function. Default export required by the Vercel platform.

```typescript
export const config = { runtime: "edge" };

export default async function handler(req: Request): Promise<Response> {
  if (req.method !== "POST") return new Response(null, { status: 405 });

  let message = "";
  try {
    const body = (await req.json()) as { message?: unknown };
    message = typeof body.message === "string" ? body.message.trim() : "";
  } catch {
    message = "";
  }
  if (!message) {
    return new Response(JSON.stringify({ error: "message required" }), {
      status: 400,
      headers: { "Content-Type": "application/json" },
    });
  }

  const apiKey = (globalThis as Record<string, string | undefined>)["RESEND_API_KEY"]
    ?? "";
  const toEmail = (globalThis as Record<string, string | undefined>)["FEEDBACK_EMAIL"]
    ?? "";
  if (!apiKey || !toEmail) {
    return new Response(JSON.stringify({ error: "server misconfigured" }), {
      status: 500,
      headers: { "Content-Type": "application/json" },
    });
  }

  const resendRes = await fetch("https://api.resend.com/emails", {
    method: "POST",
    headers: { Authorization: `Bearer ${apiKey}`, "Content-Type": "application/json" },
    body: JSON.stringify({
      from: "Hoboken Parking Feedback <onboarding@resend.dev>",
      to: toEmail,
      subject: "Hoboken Parking App Feedback",
      text: message,
    }),
  });

  if (!resendRes.ok) {
    return new Response(JSON.stringify({ error: "email delivery failed" }), {
      status: 502,
      headers: { "Content-Type": "application/json" },
    });
  }
  return new Response(JSON.stringify({ ok: true }), {
    status: 200,
    headers: { "Content-Type": "application/json" },
  });
}
```

**Note on `process.env`:** Vercel Edge Functions expose environment variables via `globalThis` (not `process.env`) at runtime. The cast above is required because TypeScript does not know about Vercel's runtime env injection; `strict: true` would reject a bare `process.env` in an Edge context.

### Tests: `tests/unit/feedback.test.ts` — new file

Environment: `node` (default). Install a minimal fake DOM on `globalThis` via `beforeEach`; remove in `afterEach`. Stub `fetch` with `vi.stubGlobal("fetch", ...)`. Use `vi.useFakeTimers()` to control `setTimeout`.

Each fake element must support: `addEventListener` (captures handler by event name), `classList` with `add/remove/contains`, `getAttribute/setAttribute`, mutable `value` (textarea), mutable `textContent` and `className` (confirm), mutable `disabled` (submit button), `focus()` (no-op).

**Test cases (all use `describe("F-28 initFeedback")`):**

- GIVEN `initFeedback` is called, WHEN `#feedback-btn` click fires, THEN `#feedback-overlay` classList contains `"open"`

- GIVEN modal is open, WHEN `#feedback-cancel` click fires, THEN `#feedback-overlay` classList does NOT contain `"open"` AND `#feedback-text` value is `""`

- GIVEN modal is open, WHEN `#feedback-overlay` click fires with `event.target` equal to the overlay element, THEN overlay classList does NOT contain `"open"`

- GIVEN modal is open, WHEN `#feedback-overlay` click fires with `event.target` equal to the inner modal element (not the overlay), THEN overlay classList still contains `"open"`

- GIVEN `#feedback-text` value is `""`, WHEN `#feedback-submit` click fires, THEN `fetch` is NOT called

- GIVEN `#feedback-text` value is `"  "` (whitespace only), WHEN `#feedback-submit` click fires, THEN `fetch` is NOT called

- GIVEN `#feedback-text` value is `"great app"`, WHEN `#feedback-submit` click fires, THEN `fetch` is called once with first arg `"/api/feedback"` AND the parsed request body equals `{ message: "great app" }` AND `#feedback-submit` disabled is `true`

- GIVEN submit fires and `fetch` resolves with `{ ok: true }`, THEN `#feedback-confirm` textContent is `"Thanks — feedback received!"` AND className is `"success"`

- GIVEN submit fires and `fetch` resolves with `{ ok: false }`, THEN `#feedback-confirm` textContent contains `"Something went wrong"` AND `#feedback-submit` disabled is `false`

- GIVEN submit fires and `fetch` rejects, THEN `#feedback-confirm` textContent contains `"Could not send"` AND `#feedback-submit` disabled is `false`
