### F-10.2 Tap to Set Position

The user taps anywhere on the map to indicate where they are. `registerMapClickHandler` (from F-07) is wired in `main.ts`. In browsing mode, a tap calls `renderPositionMarker` and stores the tapped coordinates as the current position.

**Tests**
- GIVEN the app is in browsing mode, WHEN the map click handler fires with (40.744, -74.032), THEN `renderPositionMarker` is called with those exact coordinates.
- GIVEN the map click handler fires twice at different coordinates, THEN `renderPositionMarker` is called twice — only one marker tracks the latest position.
- GIVEN no tap has occurred (userLat/userLng are null), WHEN SAVE MY SPOT is tapped, THEN `showStreetSidePicker` is not called (save is suppressed).

### F-10.3 Active Sign Pins Visible on Map

`app/map.ts` must export `signColor(reason: string): string` — a pure function returning a CSS color string for a given sign reason. `renderSignPins` uses this to color each marker.

**Tests**
- GIVEN `signColor("CONSTRUCTION")`, THEN it returns `"red"`.
- GIVEN `signColor("DELIVERY")`, THEN it returns a non-red color string (any defined non-empty string that is not `"red"`).
- GIVEN the app is in browsing mode with 3 active signs, WHEN `renderState` fires, THEN `renderSignPins` is called with an array of length 3 (mock-based: `vi.mock("../../app/map")`).
- GIVEN `app/map.ts` source, THEN `renderSignPins` contains popup-building code that includes a sign's `address` and `reason` fields.

### F-10.4 Save My Spot

**Tests**
- GIVEN the user has tapped a position, WHEN SAVE MY SPOT is tapped, THEN the street-side picker appears.
- GIVEN the user selects "N", WHEN the spot is saved, THEN the saved spot's `lat` is the tapped lat plus approximately 0.00009 degrees, and `lng` is unchanged.
- GIVEN the user selects "E", WHEN the spot is saved, THEN the saved spot's `lng` is the tapped lng plus approximately 0.00009 degrees, and `lat` is unchanged.
- GIVEN the spot is saved, THEN the app transitions to `"parked"` state.
- GIVEN the spot is saved, THEN a spot marker appears on the map and sign pins update to reflect nearby-only view.
- GIVEN the spot is saved, THEN a confirmation toast is shown briefly.
- GIVEN the app is closed and reopened after saving, THEN it opens in `"parked"` mode with the same spot (persisted in localStorage).
