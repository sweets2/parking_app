### F-13.1 isDataStale Pure Function

`isDataStale(fetchedAt: string, now: Date): boolean` in `shared/parking-logic.ts`. Returns `true` if more than 25 hours have elapsed since `fetchedAt`. Returns `true` for unparseable input (fail-safe).

Use `NOW_STABLE` from `tests/fixtures/signs.ts` as the `now` argument in all tests. Construct `fetchedAt` as an ISO string offset from `NOW_STABLE`.

**Tests**
- GIVEN `fetchedAt` is `NOW_STABLE` minus 24 hours (ISO string), WHEN `isDataStale` is called with `now = NOW_STABLE`, THEN returns `false`.
- GIVEN `fetchedAt` is `NOW_STABLE` minus 26 hours (ISO string), WHEN `isDataStale` is called with `now = NOW_STABLE`, THEN returns `true`.
- GIVEN `fetchedAt` equals `NOW_STABLE.toISOString()`, WHEN `isDataStale` is called with `now = NOW_STABLE`, THEN returns `false`.
- GIVEN `fetchedAt` is `NOW_STABLE` minus 25 hours and 1 minute (ISO string), WHEN `isDataStale` is called with `now = NOW_STABLE`, THEN returns `true`.
- GIVEN `fetchedAt` is an empty string, WHEN `isDataStale` is called, THEN returns `true` (fail-safe — unknown freshness is treated as stale).
- GIVEN `fetchedAt` is not a valid date string, WHEN `isDataStale` is called, THEN returns `true` without throwing.

### F-13.2 Stale Data UI Warning

`showStaleBanner(hoursAgo: number): void` in `app/ui.ts`. Inserts a visible banner element reading "Sign data may be outdated — last updated [N] hours ago" where [N] is `Math.round(hoursAgo)`. The banner does not prevent the app from being usable. Called from `app/main.ts` at startup if `isDataStale` returns `true`.

**Tests**
- GIVEN `showStaleBanner(30)` is called, THEN a banner element is in the DOM containing the string `"30"`.
- GIVEN `isDataStale(fetchedAt, now)` returns `false` (data is 20 hours old), THEN `showStaleBanner` is not called (verified via mock of `app/ui.ts`).
- GIVEN `showStaleBanner` has been called, THEN the `renderLoading`/`renderBrowsingMode`/`renderClearBanner` functions are not affected — they each still produce their expected DOM output without error.
